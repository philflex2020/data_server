This server monitors a S3/MinIO bucket for new Parquet files and optionally generates synthetic test data. It is used for testing the Subscriber API and DuckDB query layer without needing a running pipeline. On production, writer.cpp writes the Parquet files directly — this server is not in the data path.

Client → Server

get_status→ server
Request current S3 connection status and configuration.
{"type": "get_status"}
set_interval→ server
Change how often the server polls S3 for new Parquet files. Minimum 1 second.
{"type": "set_interval", "seconds": 5}
FieldTypeRequiredDescription
secondsintrequiredPoll interval in seconds; clamped to ≥ 1
generate→ server
Generate synthetic Parquet files directly into S3/MinIO. Useful for seeding the bucket for DuckDB query tests. Emits generate_progress events during generation, then generate_done on completion.
{
  "type":              "generate",
  "sites":             2,
  "racks_per_site":    3,
  "modules_per_rack":  4,
  "cells_per_module":  6,
  "rows_per_file":     500,
  "num_files":         1,
  "hours_ago":         1.0
}
FieldTypeRequiredDefaultDescription
sitesintoptional2Number of sites to generate data for
racks_per_siteintoptional3Racks per site
modules_per_rackintoptional4Modules per rack
cells_per_moduleintoptional6Cells per module
rows_per_fileintoptional500Rows in each generated Parquet file
num_filesintoptional1Number of Parquet files to write
hours_agofloatoptional1.0Timestamp offset — data will appear to be from this many hours in the past

Server → Client

status← server
Sent on connect and in response to get_status.
{
  "type":          "status",
  "s3_connected":  true,
  "bucket":        "battery-data",
  "endpoint":      "http://localhost:9000",
  "poll_interval": 5
}
stats← server (on each poll)
Emitted after every S3 poll with aggregate bucket statistics.
{
  "type":        "stats",
  "file_count":  1247,
  "total_rows":  45619200,
  "total_bytes": 384901120
}
FieldTypeDescription
file_countintTotal Parquet files in bucket
total_rowsintSum of row counts across all files
total_bytesintTotal bytes used in bucket
file← server (on new file detected)
Emitted for each Parquet file that appears in S3 since the last poll. Includes a sample row for quick inspection.
{
  "type":       "file",
  "key":        "project=0/site=0/2026/03/18/20260318T210000Z.parquet",
  "size":       12480,
  "rows":       1152,
  "columns":    ["timestamp", "site_id", "rack_id", "module_id", "cell_id",
                 "voltage", "current", "temperature", "soc", "soh",
                 "resistance", "capacity", "power", "project_id"],
  "sample":     {"timestamp": 1742336400.0, "site_id": 0, "voltage": 3.714, ...},
  "arrived_at": "21:00:05"
}
FieldTypeDescription
keystringS3 object key (path within bucket)
sizeintFile size in bytes
rowsintRow count in this file
columnsstring[]Column names in file
sampleobjectFirst row as column→value map
arrived_atstringLocal time when the file was first seen (HH:MM:SS)
generate_progress← server (during generate)
Emitted for each file written during a generate request.
{
  "type":  "generate_progress",
  "done":  1,
  "total": 5,
  "key":   "project=0/site=0/2026/03/18/20260318T200000Z.parquet"
}
generate_done← server (after generate completes)
Emitted once when a generate request finishes (success or error).
// Success:
{"type": "generate_done", "files_written": 5, "total_rows": 2500}

// Failure:
{"type": "generate_done", "files_written": 2, "total_rows": 1000, "error": "S3 connection timeout"}
FieldTypeDescription
files_writtenintNumber of files successfully written before completion or error
total_rowsintTotal rows written
errorstring | absentError message if generation failed; absent on success

S3 Partition Layout

Parquet files — whether written by writer.cpp or generated by this server — follow the same partition path:

s3://{bucket}/project={project_id}/site={site_id}/{YYYY}/{MM}/{DD}/{YYYYMMDDTHHMMSSz}.parquet

DuckDB uses hive partitioning to prune to the relevant project= and site= directories. See DuckDB Query Reference for query examples.
