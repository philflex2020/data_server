I2S DMA DAC variant — 5000 Hz output, 200 µs ramp steps, smooth waveform profiles

Overview

Drop-in replacement for load_controller_lora32 with a high-speed I2S DMA DAC output path. On V2 (ESP32) and LILYGO T-Display, the ramp interpolation runs inside a dedicated Core 0 task at 5000 Hz — giving 200 µs steps instead of the original ~10 ms. All WebSocket commands, profile format, OTA, OLED/TFT display, and CAN bus code are identical to the original.

What Changed vs load_controller_lora32

I2S DMA Word Format (V2 / T-Display only)

I2S_CHANNEL_FMT_RIGHT_LEFT + I2S_MODE_DAC_BUILT_IN
Each 32-bit DMA word: [ LEFT16 | RIGHT16 ]
  LEFT  → DAC2 (GPIO26, current channel)
  RIGHT → DAC1 (GPIO25, voltage channel)
Scale: dac_8bit → (uint16_t)(dac_8bit) << 8   (upper 8 bits used)

Sequence Persistence (NVS)

Every load_profile command immediately saves both channels to ESP32 NVS (non-volatile storage) using Preferences. On the next boot the sequence is restored before WiFi connects — the device comes up ready with the last profile, not in an empty state.

NVS keyTypeContents
pVCountintNumber of voltage steps saved
pVLoopboolLoop flag for voltage channel
pVStepsbytesVoltage step array (29 B × N steps)
pCCountintNumber of current steps saved
pCLoopboolLoop flag for current channel
pCStepsbytesCurrent step array (29 B × N steps)

Max 100 steps per channel — 5.8 KB total, well within NVS limits. Step names are stored as 16-char null-terminated strings. The sequence is not auto-started on boot; send start_profile to begin execution.

Sequence JSON File Format

Produced by ↓ Save in fast_load_ctrl.html. Saved to waveform-YYYY-MM-DD_HHMM.json. Reload with ↑ Load. 

{
  "version": 1,
  "blocks": {
    "b1abc": {
      "id": "b1abc",
      "name": "Pulse",
      "color": "#58a6ff",
      "steps": [
        { "vTarget": 48.0, "iTarget": 100.0, "rampMs": 500,  "holdMs": 2000 },
        { "vTarget":  0.0, "iTarget":   0.0, "rampMs": 200,  "holdMs": 1000 }
      ]
    }
  },
  "sequence": [
    { "blockId": "b1abc", "repeatCount": 3 }
  ],
  "loopSeq": false,
  "sequences": {
    "s1xyz": {
      "id":      "s1xyz",
      "name":    "Charge cycle",
      "entries": [ { "blockId": "b1abc", "repeatCount": 3 } ],
      "loopSeq": false
    }
  },
  "activeSeqId": "s1xyz",
  "colorIdx": 1
}

blocks is the block library (named reusable waveforms). Each block has N steps. Each step has independent V and I targets; ramp and hold times are shared between channels. sequence is the working playlist. sequences is the named-sequence store.

Pipeline: sequence → DMA buffer

Six stages from UI block editor to GPIO voltage on the bench. 

┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 1 — compile()  (JS, browser)                                         │
│                                                                             │
│  sequence entries × repeatCount × block.steps → flat arrays                │
│                                                                             │
│  vSteps[i] = { targetVal: step.vTarget, rampMs, holdMs }                   │
│  iSteps[i] = { targetVal: step.iTarget, rampMs, holdMs }                   │
│  meta[i]   = { blockName, repIdx, stepIdx, tStart, tEnd, vStart, vEnd, … } │
│                                                                             │
│  A 2-block sequence with repeatCounts [2, 3] and 2 steps/block produces    │
│  2×2 + 3×2 = 10 entries per channel.  Max firmware limit: 20.              │
└─────────────────────────────────────────────────────────────────────────────┘
         │ JSON over WebSocket (port 81)
┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 2 — sendProfile()  (JS, browser)                                     │
│                                                                             │
│  Maps compile() output to firmware field names in ONE message:              │
│    targetVal → value                                                        │
│    holdMs    → durationMs                                                   │
│    adds name: "step1", "step2", …                                          │
│                                                                             │
│  { cmd: "load_profile",                                                     │
│    voltage: [{name,value,durationMs,rampMs}, …],   // ≤ 20 entries        │
│    current: [{name,value,durationMs,rampMs}, …] }                          │
└─────────────────────────────────────────────────────────────────────────────┘
         │
┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 3 — loadProfileFromJson()  (firmware, Core 1)                        │
│                                                                             │
│  portENTER_CRITICAL(&profileMux)                                            │
│    profileVRunning = false; profileCRunning = false;                        │
│    populate profileV[0..N-1] and profileC[0..N-1]                          │
│      .value, .durationMs, .rampMs, .name                                   │
│    profileVCount = N; profileCCount = M;                                    │
│  portEXIT_CRITICAL(&profileMux)                                             │
│                                                                             │
│  Followed by start_profile to set profileVRunning / profileCRunning = true. │
└─────────────────────────────────────────────────────────────────────────────┘
         │ mutex-protected read on each DMA buffer boundary (every 12.8 ms)
┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 4 — dacTask()  (firmware, Core 0, priority 2)                        │
│                                                                             │
│  Runs a tight for(;;) loop filling 64-sample DMA buffers:                  │
│                                                                             │
│  dt_ms = 1000 / 5000 = 0.2 ms per sample                                   │
│                                                                             │
│  Per sample (i = 0..63):                                                    │
│    elapsed_ms += 0.2                                                        │
│    if in_ramp:                                                              │
│      p = elapsed_ms / rampMs         (0.0 → 1.0)                           │
│      now = start + (target − start) × p                                     │
│    else (holding):                                                          │
│      if elapsed_ms ≥ durationMs:                                            │
│        step = (step + 1) % count     ← loops profile continuously          │
│        reload start/target/rampMs/holdMs from profileV[step]               │
│        in_ramp = (rampMs > 0)                                               │
│                                                                             │
│  Voltage and current ramps are independent — different rampMs/durationMs   │
│  per channel are fully supported. Both channels computed per sample.       │
│                                                                             │
│  At end of each buffer (i == 63), push lv_now → voltageValue, dac1Value.   │
└─────────────────────────────────────────────────────────────────────────────┘
         │
┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 5 — I2S word packing                                                 │
│                                                                             │
│  12-bit dac values (0–4095) → 8-bit DAC:                                   │
│    v8 = dac1Value >> 4    (drop 4 LSBs)                                     │
│    c8 = dac2Value >> 4                                                      │
│                                                                             │
│  Stereo 32-bit word format:                                                 │
│    bits [31:24]  LEFT  high byte → DAC2 / GPIO26 / current                 │
│    bits [23:16]  LEFT  low byte  → 0                                       │
│    bits [15: 8]  RIGHT high byte → DAC1 / GPIO25 / voltage                 │
│    bits [ 7: 0]  RIGHT low byte  → 0                                       │
│                                                                             │
│    buf[i] = ((uint32_t)c8 << 24) | ((uint32_t)v8 << 8)                    │
│                                                                             │
│  4 DMA buffers × 64 words × 4 bytes = 1024 bytes total DMA memory.         │
│  i2s_write() blocks until hardware DMA picks up the buffer (~12.8 ms).     │
│  Write rate: 5000 ÷ 64 ≈ 78.1 buffers/sec  (shown as bufsPerSec in status) │
└─────────────────────────────────────────────────────────────────────────────┘
         │
┌─────────────────────────────────────────────────────────────────────────────┐
│  Stage 6 — ESP32 I2S / DAC hardware                                         │
│                                                                             │
│  I2S peripheral in I2S_MODE_DAC_BUILT_IN mode drives the ESP32's two       │
│  8-bit R-2R DACs directly from DMA — no CPU intervention per sample.       │
│  DAC output range: 0 V (code 0) → ~3.3 V (code 255).                       │
│  External scaling: resistor divider / op-amp to reach 0–60 V / 0–200 A ref │
└─────────────────────────────────────────────────────────────────────────────┘
