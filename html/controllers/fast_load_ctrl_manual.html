1. Serial commissioning (/dev/ttyUSBx)

On first power-up the board has no WiFi credentials. You must provision it via the USB serial port before connecting over the network.

1.1 Find the port

Plug the LoRa32 V2 into a USB port and run:

ls /dev/ttyUSB* /dev/ttyACM*
/dev/ttyUSB0

If you see multiple entries, unplug the board and run the command again to see which one disappears.

1.2 Add user to dialout group

If you get a permission denied error on the port, add your user to the dialout group (log out and back in after):

sudo usermod -aG dialout $USER
# apply immediately without logging out:
newgrp dialout

1.3 Open the terminal

# Ctrl-A K to exit (or Ctrl-A : then quit)
screen /dev/ttyUSB0 115200
# or with picocom (Ctrl-A Ctrl-X to exit):
picocom /dev/ttyUSB0 --baud 115200 --echo --imap lfcrlf
# or minicom (Ctrl-A X to exit):
minicom -b 115200 -D /dev/ttyUSB0 -c on

Press the Reset (RST) button on the board — top-right edge as shown:

2. WiFi setup via serial

The board stores up to 4 SSID/password pairs in NVS flash and tries them in order at boot.

2.1 Add a network

wifi add MyNetwork MyPassword123
Added [0] MyNetwork

To add a second network (fallback):

wifi add OfficeWifi office_pass99
Added [1] OfficeWifi

2.2 Check connection status

wifi status
WiFi: CONNECTED  SSID: MyNetwork  IP: 192.168.86.55

Note the IP address — you will need it to connect the web UI.

2.3 List stored credentials

wifi list
WiFi credentials (2/4):
  [0] MyNetwork <active>
  [1] OfficeWifi

2.4 Remove a credential

wifi remove 1
Removed [1] OfficeWifi

2.5 Clear all credentials

wifi clear
All WiFi credentials cleared.
⚠ Credentials survive reboot
WiFi credentials are stored in NVS (non-volatile) flash. They persist across power cycles. Use wifi clear before handing the board to another person or site.
✓ Automatic reconnect
If WiFi drops, the board retries every 10 seconds, cycling through all stored credentials. No reboot needed.

3. Connecting with the web UI

3.1 Open the page

Open fast_load_ctrl.html directly in a browser — no web server needed. Double-click the file in your file manager, or drag it onto a browser window, or:

xdg-open fast_load_ctrl.html   # Linux
open fast_load_ctrl.html        # macOS

The page runs entirely in the browser. It connects to the device over WebSocket once you enter the IP address — the HTML file itself can be on any drive or directory.

3.2 Enter the device address

  1. In the connection bar at the top, enter the IP address shown by wifi status in the Host field.
  2. Leave port at 81 (WebSocket).
  3. Click Connect.

The indicator dot turns green and the status bar shows Connected. The WS Log panel at the bottom will show the handshake messages.

3.3 Verify with Status

Click the Status button in the connection bar. The WS Log will show the current voltage, current, profile state and WiFi info reported by the firmware.

3.4 System Scale

Below the connection bar is a System Scale bar. Set:

  • Max V — the maximum voltage your hardware outputs (default 60 V, range limited by hardware to 60 V)
  • Max A — the maximum current (default 200 A)

All waveform sliders and preset values are scaled to these limits. Set them before creating blocks.

ℹ DAC range
The LoRa32 V2 outputs 0–3.3 V on GPIO25/GPIO26. With the LM324 amplifier stage (gain 3.485×) the output range is 0–11.5 V ≈ 12 V. The firmware maps Max V / Max A to DAC full-scale — set these to match your transducer/load scaling.

4. Block Library — creating waveform blocks

A block is a named multi-step waveform. Each step defines a voltage target, current target, ramp time (ms) and hold time (ms). Blocks are the building material for sequences.

4.1 Create a new block

1In the Block Library panel (left column), click + New.
2The block editor expands. Type a name in the Block Name field (e.g. Ramp Up).
3Click + Step to add a step. Fill in the columns:
ColumnUnitsDescription
V (V)VoltsTarget voltage at end of ramp
I (A)AmpsTarget current at end of ramp
Ramp msmsTime to ramp from previous value to this target (0 = instant)
Hold msmsTime to hold at this value before moving to next step
4Add more steps as needed. Use the ↑ ↓ row buttons to reorder, × to delete a step.
5Click Save to commit the block to the library.

The block appears in the library list. The Waveform Preview canvas (top-right) shows voltage and current traces for the selected block. Toggle the V Voltage / I Current buttons to show/hide each channel.

4.2 Edit an existing block

Click the block card in the library list. The editor opens with that block's data. Make changes and click Save. To discard, click ✕.

4.3 Delete a block

Open the block editor and click Delete. If the block is used in a sequence you will be warned first.

4.4 Load a preset

The — Load Preset — dropdown in the connection bar contains built-in waveform examples (charge pulse, constant current ramp, etc.). Select one and click Load to populate the block library with that preset's blocks.

5. Sequences — building a playlist

A sequence is an ordered list of blocks with optional repeat counts and a loop toggle. Multiple named sequences can be stored locally in the browser.

5.1 Build a sequence

  1. In the Sequence panel, use the — add block — dropdown at the bottom to select a block from the library.
  2. Click Add. The block appears in the sequence list with a repeat count of 1.
  3. Repeat to add more blocks in order.
  4. Change the rpt number on any entry to repeat that block N times within the sequence.
  5. Use the ↑ ↓ arrows on each entry to reorder. Click × to remove.
  6. Toggle Loop on/off to make the entire sequence repeat continuously.

5.2 Save a named sequence

  1. Click + New in the sequence name bar. Enter a name (e.g. Morning Cycle).
  2. The current sequence is saved under that name in browser local storage.

5.3 Load a named sequence

Select a name from the sequence dropdown (— select sequence —). The sequence list is immediately populated.

5.4 Rename or delete a saved sequence

  • Rename — select it, click Rename, enter new name.
  • Del — select it, click Del. Does not affect the blocks in the library.

6. Running, stopping and monitoring

6.1 Load a profile to the device (without starting)

Click ↑ Load (no start) in the Playback panel. The current sequence is serialised and sent to the firmware over WebSocket. The device stores it but does not begin outputting. Use this to pre-arm a profile before a test.

6.2 Start a sequence

  1. Choose the output channels from the Both Channels / Voltage Only / Current Only selector.
  2. Click ▶ Start. The sequence is sent and playback begins immediately.

The Playback panel shows live status:

IndicatorDescription
BlockName of the block currently playing
StepCurrent step index within that block
Repeat / CycleCurrent repeat count and sequence cycle number
FW: V | CFirmware-reported step index for voltage and current — independent cross-check
Progress barEstimated progress through the current step hold time
TimelineColour-coded strip showing relative block durations in the sequence

6.3 Stop a sequence

Click ■ Stop. The firmware immediately halts both channels. DAC outputs are held at their last value until overridden.

⚠ Outputs are not zeroed on stop
profile stop halts profile stepping but holds the last DAC value. To return outputs to zero: use set voltage 0 and set current 0 in the WS Terminal, or issue those commands via serial.

6.4 Direct voltage / current override

While connected you can send immediate one-shot commands from the WS Log area (type in the input bar below the log) or via serial:

# Set voltage to 24 V immediately
set voltage 24

# Set current to 50 A immediately
set current 50

# Raw DAC (0-4095) — bypasses scaling
set dac1 2048
set dac2 1024

7. Save and restore schedules

Blocks and sequences live in browser local storage and are also exportable as JSON files for backup and transfer between machines.

7.1 Save to a JSON file

In the Block Library panel header, click ↓ Save. Your browser downloads a .json file containing all blocks and all named sequences.

7.2 Load from a JSON file

Click ↑ Load in the Block Library header and select a previously saved .json file. All blocks and sequences in the file are merged into the current library.

✓ Tip: version your schedule files
Name files with a date, e.g. fastload_site_A_2026-04-08.json. The same file can be loaded on any machine running the UI without reconnecting to the device.

7.3 What is stored where

DataWhere storedPersists across…
Blocks & named sequencesBrowser local storage + JSON exportBrowser sessions; survives page reload
Last loaded profile (steps)Device NVS flashDevice power cycles; auto-starts on boot if it was running when saved
WiFi credentialsDevice NVS flashDevice power cycles, firmware reflash does NOT clear

ℹ Auto-start on power cycle
When you click ▶ Start, the firmware saves the active profile to NVS and marks it as running. On next boot the device restores and auto-starts that profile. To prevent auto-start: click ■ Stop before removing power (this saves the profile with stopped state to NVS).

7.4 Modify an existing schedule

  1. Click a block in the library to open the editor.
  2. Change step values, add or remove steps.
  3. Click Save.
  4. If the modified block is part of a named sequence, select that sequence — it uses the updated block automatically (sequences reference blocks by ID).
  5. Click ▶ Start to send the updated profile to the device.

9. WebSocket interface protocol

The device listens on port 81. All messages are UTF-8 JSON text frames. When a secret is set, the device issues a challenge on connect and all commands are blocked until the client authenticates (see §10). With no secret set the device sends {"type":"ok","msg":"authenticated"} immediately and commands are accepted.

9.1 Commands — client → device

Every command has a "cmd" string field. Additional fields depend on the command.

9.3 load_profile message

Voltage and current steps are independent arrays. Each step:

// step object fields:
{
  "name":       "ramp-up",  // label (display only)
  "value":      12.5,       // target value (V or A)
  "durationMs": 2000,       // total hold time at target
  "rampMs":     500         // linear ramp time to reach target (0 = instant)
}

Full example — two voltage steps, no current profile:

{
  "cmd":   "load_profile",
  "loop":  true,
  "voltage": [
    {"name":"low",  "value":5.0,  "durationMs":1000, "rampMs":200},
    {"name":"high", "value":12.0, "durationMs":1000, "rampMs":200}
  ]
}
Max 100 steps per channel. Up to 16 KB JSON payload accepted.

10. Security — challenge-response authentication

The firmware uses HMAC-SHA256 challenge-response to authenticate WebSocket clients. The secret is never sent over the network — only a one-time signed proof that the client knows it.

10.1 How it works

  1. On connect, the device generates a fresh 32-char random hex nonce and sends it as a challenge.
  2. The client computes HMAC-SHA256(secret, nonce) and sends the 64-char hex result in an auth command.
  3. The device computes the same HMAC internally. If they match, it replies {"type":"ok","msg":"authenticated"} and unlocks the connection.
  4. The nonce is unique per connection — replaying a captured auth response does not work.
// 1. Device → client (on connect)
{"type":"challenge","nonce":"bd16ff69a655584b320eba2645c459a1"}

// 2. Client → device
{"cmd":"auth","response":"a3f7c2e1..."}   // HMAC-SHA256(secret, nonce)

// 3. Device → client
{"type":"ok","msg":"authenticated"}

10.2 Setting the secret — first-time setup

After flashing, the NVS (flash storage) is blank and the device boots in open-access mode — no challenge is sent and all commands are accepted immediately.

Option A — set via serial console (recommended):

set secret mypassword
WS secret updated (10 chars)

Option B — set via web UI:
Connect with fast_load_ctrl.html while the device is in open-access mode. A ⚠ No secret set bar appears. Type your secret and click Lock Device. The set_secret WS command is accepted once (when no secret is currently set) and then locked.

✓ Auto-fill after locking
The web UI copies the new secret into the auth field automatically, so the next reconnect authenticates without retyping.

10.3 Changing the secret

Once a secret is set, set_secret over WebSocket is rejected with an error. Use the serial console — physical access required:

set secret newpassword   // change secret
set secret               // clear secret — re-enables open-access mode

10.4 Open-access mode

If wsSecret is empty in NVS, the device skips the challenge entirely and sends {"type":"ok","msg":"authenticated"} as the first message. The fast_load_ctrl_open.html page is designed for this mode — it listens for that message before sending commands.

⚠ Open-access is LAN-only
Open-access mode is suitable for trusted LANs only. Set a secret before deploying on any shared or internet-accessible network.

10.5 Multiple clients

Up to 4 WebSocket clients can be connected simultaneously. Each connection has its own nonce and authentication state — one client authenticating does not authenticate others.
