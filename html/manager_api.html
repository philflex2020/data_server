Manager API

Process manager WebSocket — start, stop, and monitor all pipeline components

Protocol: WebSocket (JSON)
Port: 8760
URL: ws://<host>:8760
Source: manager/manager.py
Config: manager/config.yaml
TTD Hub Manager Dashboard Generator API Stress Runner API AWS Server API
The manager controls all pipeline processes defined in config.yaml. Each component runs as a subprocess with stdout/stderr streamed to connected WebSocket clients. On phil-dev the manager listens on :8761; on spark-22b6 / .51 on :8762.

Client → Server

get_status→ server
Request current status of all registered components.
{"type": "get_status"}
start→ server
Start a single component by name. Component name must match a key in config.yaml → components.
{"type": "start", "component": "generator"}
FieldTypeRequiredDescription
componentstringrequiredComponent name from config (e.g. generator, parquet_writer, subscriber_api)
stop→ server
Stop a single running component. SIGTERM is sent; process is waited on.
{"type": "stop", "component": "generator"}
FieldTypeRequiredDescription
componentstringrequiredComponent name to stop
start_all→ server
Start all components in the order defined by config.yaml → order.
{"type": "start_all"}
stop_all→ server
Stop all running components in reverse start order.
{"type": "stop_all"}
rescan→ server
Adopt processes already running on the host that match component definitions (by command name match). Useful after a manager restart when pipeline processes are still alive.
{"type": "rescan"}

Server → Client

status← server
Sent immediately on connect and after any start/stop operation. Contains state of all components.
{
  "type": "status",
  "components": {
    "generator":       {"status": "running", "pid": 14480, "label": "Data Generator"},
    "parquet_writer":  {"status": "stopped", "pid": null,  "label": "Parquet Writer (NATS → S3)"},
    "subscriber_api":  {"status": "error",   "pid": null,  "label": "Subscriber API (NATS + DuckDB)"}
  }
}
FieldTypeDescription
componentsobjectMap of component name → component state
components[name].status"running" | "stopped" | "error"Current lifecycle state
components[name].pidint | nullOS process ID; null when not running
components[name].labelstringHuman-readable label from config
log← server
Streamed in real time as a component writes to stdout or stderr.
{
  "type":      "log",
  "component": "parquet_writer",
  "line":      "[flush] 1152 rows → project=0/site=0/2026/03/18/20260318T210000Z.parquet",
  "stream":    "stdout"
}
FieldTypeDescription
componentstringComponent that produced the line
linestringLog line (newline stripped)
stream"stdout" | "stderr"Which output stream

Configuration Reference

Manager config lives in manager/config.yaml. The order list controls start sequence; each entry is a key in components.

websocket:
  host: 0.0.0.0
  port: 8760          # change per machine (8761 phil-dev, 8762 spark)

order:
  - flashmq           # or mosquitto
  - nats              # if using NATS pipeline
  - generator
  - parquet_writer
  - subscriber_api

components:
  generator:
    label: "Data Generator"
    cmd:   [".venv/bin/python", "generator.py", "--config", "config.yaml"]
    cwd:   "source/generator"

  parquet_writer:
    label: "Parquet Writer (MQTT → Parquet)"
    cmd:   ["source/parquet_writer_cpp/parquet_writer",
            "--config", "source/parquet_writer_cpp/config.spark.yaml"]
    cwd:   "."
    ports: []

Example: Connecting and Monitoring

const ws = new WebSocket("ws://192.168.86.51:8760");

ws.onopen = () => ws.send(JSON.stringify({type: "get_status"}));

ws.onmessage = ({data}) => {
  const msg = JSON.parse(data);
  if (msg.type === "status") {
    for (const [name, state] of Object.entries(msg.components))
      console.log(name, state.status, state.pid);
  }
  if (msg.type === "log")
    console.log(`[${msg.component}] ${msg.line}`);
};

// Start the writer
ws.send(JSON.stringify({type: "start", component: "parquet_writer"}));
