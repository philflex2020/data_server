site.yaml — Documentation Needed TODO

The site.yaml query: section has grown significantly. DevOps need to know what to set for a new deployment. The editor validates it but the keys are not documented end-to-end anywhere except code comments.

Keys that need prose docs

KeySectionWhat to documentStatus
query.hot_base query: Path to hot-tier rg3 parquet files; used by all Flux queries Missing
query.wide_base query: Path to compact wide files (convert_wide_v2 output); optional, enables cold-range routing Missing
query.influx_compat query: true → emit _measurement/_field/_value columns so Grafana InfluxDB dashboards work Missing
query.tag_map query: Rename columns for Grafana: e.g. unit_id → unit_controller_id to match InfluxDB tags Missing
query.file_glob query: Glob pattern for parquet files within date dirs; default *.parquet Missing
query.live_connect query: Arrow IPC address of writer live stream; e.g. tcp://localhost:8774 Missing
query.hot_days query: Number of days to keep in hot tier before routing to cold/wide; default 7 Missing
capture.host_path capture: Node path for PVC; must match storageClassName: local-path In CLAUDE.md
eks.storage_class eks: Must be local-path on k3s — gp2 leaves PVC Pending In CLAUDE.md
Suggested location: add a docs/site-yaml-reference.md page to the mkdocs site on .23. The editor's JSON schema already serves as machine-readable docs; the markdown page should cover the "why" — when to set each key and what goes wrong if you don't.

Suggested deploy sequence for DevOps hand-off

# 1. Edit site.yaml for target site
vi src/site.yaml   # set mqtt.host, site.id, capture.host_path, eks.storage_class

# 2. Regenerate manifests
cd src && make configmap

# 3. Build images (or pull from registry)
make image        # parquet-writer:latest
make query-image  # parquet-query:latest

# 4. Import to k3s and deploy
make demo         # writer: import + apply + rollout
make query-demo   # query: import + apply + rollout

# 5. Smoke test
curl http://<node>:8771/health   # writer healthy + mqtt_connected
curl http://<node>:8772/health   # query healthy + live_connected + live_series > 0
curl http://<node>:8772/sites    # returns unit IDs seen in parquet files

# 6. Grafana — add InfluxDB v2 datasource
#   URL: http://<node>:8772
#   Org: ems  ·  Bucket: ems  ·  Token: (any non-empty string)
