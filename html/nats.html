Monitoring API Endpoints


All endpoints are plain HTTP GET on port 8222. No auth required (internal network only).




  /healthz
  Server health check — returns {"status":"ok"} if running
  /healthz ↗




  /varz
  General server variables — version, uptime, connections, message & byte rates, config summary
  /varz ↗




  /jsz
  JetStream summary — stream count, consumer count, memory & file store totals
  /jsz ↗




  /jsz?streams=true&consumer=true
  JetStream with full stream config + per-stream consumer list and delivery state
  /jsz?streams=true&consumer=true ↗




  /connz
  Client connections — name, IP, subscriptions, message & byte counts per client
  /connz ↗




  /subsz
  Subscription routing table — total subjects, matches, slow consumers
  /subsz ↗





FlashMQ Monitoring



  FlashMQ (the MQTT broker on port 1883) does not expose an HTTP monitoring API.
  On spark-22b6 it runs as an OS-level service (systemd), not via the process manager.
  Monitoring options:



    
  
  • Process status via process manager WebSocket (:8762) — shows running/stopped for the stack
    • 
  
  • 
    FlashMQ publishes broker statistics to $SYS/# MQTT topics every 10 s, including:
    
    $SYS/broker/clients/connected — active client count
    
    $SYS/broker/messages/publish/received — total publishes received
    
    $SYS/broker/messages/publish/sent — total publishes forwarded
    
    $SYS/broker/bytes/received and $SYS/broker/bytes/sent
  
    • 
  
  • NATS message rate in /varz above reflects the FlashMQ → bridge throughput indirectly
    • 
  
  • Log file: /var/log/flashmq/flashmq.log (info level by default)
    • 
  
  • Health check: mosquitto_sub -t '$SYS/broker/uptime' -C 1
    •
