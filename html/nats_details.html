TEST 5 Schema Flexibility

Zero-config schema evolution. The parquet writer collects all keys from all rows dynamically (all_keys = {k for r in flat for k in r}). New fields appear automatically as new Parquet columns. Missing fields become NULLs — the record is always written.
ScenarioTelegraf / InfluxDBNATS + Parquet
Extra field in payload Silently ignored — not in config mapping Automatically stored as new Parquet column
Missing field (e.g. power) Error → entire message DROPPED, 0 writes NULL in that column — record preserved
Nested {"value": X, "unit": "V"} payload Error → entire message DROPPED _flatten() handles both formats transparently
New source type (solar, wind) Not subscribed — telegraf only listens to batteries/# Subject filter > catches all subjects automatically
Schema change Edit config + restart telegraf required Zero config, zero restart, zero downtime
Parse error recovery Manual restart — missed data lost forever JetStream replay from last acknowledged position
