Hot file advantage. Queries Q1, Q4, Q5, Q6 target hot.parquet — a single flat file containing all partitions from the last flush (~14 976 rows, ~60 s lag). DuckDB reads this in one pass with no S3 list-objects overhead. Column projection and predicate pushdown reduce actual bytes read further.
Q2 / Q3 — date-scoped glob paths. Rather than scanning all historical files with /**/*.parquet, queries construct glob paths covering only the date directories that overlap the requested time window (e.g. /2026/03/17/*.parquet for a same-day query). This drops Q2/Q3 from ~300 ms (full store) to ~55 ms by limiting DuckDB to 356 files instead of 2 640 — even at the 10 s test flush rate. In production with a 60 s flush: ~60 files/day/site, expected < 20 ms.
With the AWS writer in place, the host flush interval can be extended to 10–15 minutes. current_state.parquet on AWS covers the real-time gap — the partitioned S3 store only needs to satisfy historical queries, not current-state lookups. At a 15-minute flush: ~96 files/site/day instead of 1 440 (at 1 min). Date-scoped Q2/Q3 latency scales roughly with file count — projecting ~15 ms locally, ~135 ms on S3. Larger files also compress better and give DuckDB more accurate row-group statistics for predicate pushdown. Set flush_interval_seconds: 900 on the host writer once AWS writer is deployed.

No per-query charge. DuckDB reads Parquet directly from S3 at $0.0004/GET. A 6-query cycle every 10 s = 2 160 GETs/hour ≈ 1.6 M GETs/month ≈ $0.64/month total S3 API cost, independent of fleet scale. Compare to InfluxDB Cloud at $28.73/month at 24× scale.
Partition pruning halves scan cost for site-specific queries. The directory structure site={id}/YYYY/MM/DD/ lets DuckDB skip all other sites' files via Hive partitioning. Q2 and Q3 with hive_partitioning=true read only 1/48 of the total parquet store.

Query Details

Q1 Latest snapshot — all battery cells, site 0 (last 5 min) hot.parquet

Web dashboard: 'current status' panel — last reading per cell. Uses hot.parquet for sub-90s recency; falls back to partitioned store for older data.

-- Local: use hot.parquet for freshest data (atomic, written every flush)
SELECT site_id, rack_id, module_id, cell_id,
       arg_max(voltage,     timestamp) AS voltage,
       arg_max(current,     timestamp) AS current,
       arg_max(temperature, timestamp) AS temperature,
       arg_max(soc,         timestamp) AS soc,
       arg_max(soh,         timestamp) AS soh
FROM read_parquet('/srv/data/parquet/hot.parquet')
WHERE timestamp > epoch(now()) - 300
  AND site_id = 0
GROUP BY ALL
ORDER BY rack_id, module_id, cell_id

-- S3: combine hot file + recent partitioned files
SELECT site_id, rack_id, module_id, cell_id,
       arg_max(voltage, timestamp) AS voltage,
       arg_max(current, timestamp) AS current,
       arg_max(temperature, timestamp) AS temperature,
       arg_max(soc, timestamp) AS soc,
       arg_max(soh, timestamp) AS soh
FROM read_parquet([
    's3://bucket/parquet/batteries/project=0/site=0/**/*.parquet',
    's3://bucket/parquet/hot.parquet'
], hive_partitioning=true, union_by_name=true)
WHERE timestamp > epoch(now()) - 300
GROUP BY ALL
ORDER BY rack_id, module_id, cell_id

Q2 Time series — single cell history (last hour, date-scoped) partitioned

Web dashboard: 'cell detail' chart — voltage/temperature trend for one cell. Date-scoped glob limits files opened to those in the relevant date directory.

-- Construct the path for today only (app-level, Python/JS generates YYYY/MM/DD)
SELECT timestamp, voltage, current, temperature, soc
FROM read_parquet(
    '/srv/data/parquet/batteries/project=0/site=0/2026/03/17/*.parquet'
)
WHERE timestamp > epoch(now()) - 3600
  AND rack_id   = 0
  AND module_id = 0
  AND cell_id   = 0

Q3 Rack aggregates — mean voltage & temperature per rack (10s buckets, last hour) partitioned

Web dashboard: 'rack health' heatmap — mean per rack per 10-second bucket, all sites. Date-scoped across all site directories to limit files opened.

-- Date-scoped: scan today's files across all sites (app constructs YYYY/MM/DD)
SELECT site_id, rack_id,
       time_bucket(INTERVAL '10 seconds', to_timestamp(timestamp)) AS bucket,
       round(avg(voltage),     3) AS mean_voltage,
       round(avg(temperature), 2) AS mean_temp,
       round(avg(soc),         2) AS mean_soc
FROM read_parquet(
    '/srv/data/parquet/batteries/project=0/*/2026/03/17/*.parquet',
    union_by_name=true
)
WHERE timestamp > epoch(now()) - 3600
GROUP BY ALL
ORDER BY bucket DESC, site_id, rack_id

Q4 Alert scan — cells with voltage outside 3.2–4.1 V (last 5 min) hot.parquet

Web dashboard: alert panel — cells outside safe voltage window across the entire fleet. Hot file gives sub-90s recency without full partitioned scan.

Q5 Solar overview — last irradiance & AC power, all panels (last 5 min) hot.parquet

Web dashboard: 'solar fleet' overview — last reading per panel across all sites. InfluxDB took 236 ms for this at 1× scale; hot.parquet gives a single-file scan.

-- hot.parquet contains both batteries and solar rows (union_by_name handles sparse columns)

Q6 Fleet health — active cell count & data freshness (last 2 min) hot.parquet

Web dashboard: 'system health' badge — confirm data is flowing from all sites and both source types. Uses hot.parquet for the lowest possible lag.

SELECT source_type,
       site_id,
       count(*)                   AS readings,
       max(timestamp)             AS latest_ts,
       round(epoch(now()) - max(timestamp), 1) AS age_seconds
FROM (
    -- batteries
    SELECT 'batteries' AS source_type, site_id, timestamp
    FROM read_parquet('/srv/data/parquet/hot.parquet')
    WHERE voltage IS NOT NULL
    UNION ALL
    -- solar
    SELECT 'solar' AS source_type, site_id, timestamp
    FROM read_parquet('/srv/data/parquet/hot.parquet')
    WHERE irradiance IS NOT NULL
)
WHERE timestamp > epoch(now()) - 120
GROUP BY ALL
ORDER BY source_type, site_id

Data Model Reference

Schema — batteries

-- Parquet columns in /srv/data/parquet/batteries/…/*.parquet
project_id  INT64    -- from config: output.project_id
site_id     INT64    -- from topic: batteries/site={n}/…
rack_id     INT64    -- from topic: …/rack={n}/…
module_id   INT64    -- from topic: …/module={n}/…
cell_id     INT64    -- from topic: …/cell={n}
timestamp   DOUBLE   -- unix epoch seconds (float, from JSON payload)
voltage     DOUBLE   -- V
current     DOUBLE   -- A  (negative = charging)
temperature DOUBLE   -- °C
soc         DOUBLE   -- %  state of charge
soh         DOUBLE   -- %  state of health
resistance  DOUBLE   -- Ω
