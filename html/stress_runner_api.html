Stress Runner API

24/7 load generator WebSocket — read-only stats on sustained MQTT publishing

Protocol: WebSocket (JSON)
Port: 8769
URL: ws://<host>:8769
Source: source/stress_runner/stress_runner.py
Config: source/stress_runner/config.yaml
MQTT target: localhost:1883 (FlashMQ / Mosquitto)
TTD Hub Stress Runner Dashboard Manager API Generator API
The stress runner is a read-only API — it cannot be started or stopped via WebSocket. It runs 24/7 on spark-22b6 / .51 and broadcasts stats every second to all connected clients. To start or stop it, use the Manager API.
Default topology
1,152
cells (1 proj × 3 sites × 4 racks × 8 mod × 12 cells)
Sustained rate
~1,152
msg/s at 1 loop/s/site
Task model
asyncio
one task per (project, site)

Client → Server

get_status→ server
Request an immediate stats reply. The server also broadcasts stats automatically every second, so this is only needed on initial connect if you cannot wait for the next broadcast.
{"type": "get_status"}

Server → Client

stats← server (broadcast every 1 s)
Emitted every second to all connected clients. Also sent immediately on connect and in response to get_status.

MQTT Output Format

Topic format

{topic_prefix}/project={project_id}/site={site_id}/rack={rack_id}/module={module_id}/cell={cell_id}
# Example:
batteries/project=0/site=0/rack=1/module=2/cell=3
Unlike the generator, the stress runner always includes project= in the topic. This is required by writer.cpp's default topic_kv_map when project_id is used as a partition field.

Payload — flat format

{
  "timestamp":   1741234567.891,
  "project_id":  0,
  "site_id":     0,
  "rack_id":     1,
  "module_id":   2,
  "cell_id":     3,
  "voltage":     3.714,
  "current":    -0.42,
  "temperature": 28.3,
  "soc":         71.2,
  "soh":         97.8,
  "resistance":  0.0118,
  "capacity":    99.4,
  "power":      -1.560
}

power is a derived field — computed as voltage × current when both are present in the schema.

Configuration Reference

mqtt:
  host:      localhost
  port:      1883          # FlashMQ on spark/.51
  client_id: stress-runner

sample_interval_seconds: 1   # one full topology publish per second per (project, site) task

topic_prefix: batteries      # root of MQTT topic hierarchy

projects:
  - project_id: 0
    sites:
