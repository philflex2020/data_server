FAILURE MODE 1  Nested Payload — Error Suppression

Telegraf detected JSON paths that don't exist (e.g. soh as a nested object instead of a flat scalar). The error was logged 100800 time(s) then suppressed. All subsequent messages fail silently — 0 writes, 0 metrics_dropped increment.

Silent data loss. Once suppressed, no log entry, no metric, no Grafana gap indicator. Fix: ensure generator uses payload_format: flat, then restart telegraf.

FAILURE MODE 2  QoS Queue Overflow → Consumer Stall

FlashMQ logged 12118 QoS drop warning(s) for telegraf-battery-consumer. After this event, the Telegraf MQTT consumer may stall — the messages_received counter freezes while Telegraf appears healthy (connected, writing internal metrics).

Self-healing: No. Only sudo systemctl restart telegraf recovers the consumer. Observable only by watching messages_received in internal metrics — not visible in Grafana without a custom alert.
