InfluxDB line protocol overhead: the full topic string (unit/02281660/bms/bms_1/Batt1_MBMS_…/integer, avg 84.4 bytes) repeats in every single line protocol message. At 81,420 msg/s this alone contributes 6.9 MB/s of wire data — before field names, measurement name, or timestamp. Parquet eliminates this through dictionary encoding: 1,770 unique point names are stored once per row group (~106 KB total) regardless of how many millions of rows reference them.

6 · S3 Lifecycle — Recommended Configuration

S3 lifecycle rules can move Parquet files to cheaper tiers automatically after a configurable age. DuckDB and Athena can query S3-IA files with identical syntax; the only difference is a small per-GB retrieval fee on cold data. 

# S3 lifecycle policy (JSON / Terraform)
{
  "Rules": [{
    "ID":     "parquet-tiering",
    "Status": "Enabled",
    "Filter": { "Prefix": "parquet/" },
    "Transitions": [
      { "Days": 30,  "StorageClass": "STANDARD_IA"  },   // -46% vs Standard
      { "Days": 365, "StorageClass": "GLACIER_IR"   }    // -82% vs Standard ($0.004/GB)
    ]
  }]
}

With Glacier Instant Retrieval for data older than 1 year, the effective blended storage rate across 10 years drops below $0.010/GB-month on average. Athena and DuckDB access Glacier IR at the same speed as S3 Standard — there is no restore step. The $0.03/GB retrieval fee only matters if you scan the entire 10-year history frequently; partitioned (per-unit) queries typically scan <1% of the total dataset per query.

7 · Why the Cost Gap Is So Large

FactorInfluxDB/TelegrafParquet/DuckDB
String storage model Full topic string (avg 84 bytes) written in every line protocol message. Repeated 81,420 times/sec = 6.9 MB/s of string overhead before values. Dictionary encoding: 1,770 unique point names stored once per row group (~106 KB). Every row stores a 2-byte index. String length has zero per-row cost.
Storage medium EBS gp3: $0.08/GB-month. Necessary for InfluxDB's random-read TSM query engine — cannot use S3 directly (OSS edition). S3: $0.023/GB-month (3.5× cheaper). Parquet is a scan-optimised columnar format — works natively with object storage.
On-disk size ~123 GB/day after 8× TSM compression. TSM block indices embed the full series key (tag strings) in every 1,000-row block header. ~67 GB/day after Parquet snappy compression. No per-row metadata; column statistics compressed with RLE and bit-packing.
Query scalability Full table scan unless a matching tag index exists. Adding new tag combinations requires re-ingestion or separate continuous queries. DuckDB/Athena prune files by directory (per-unit partitioning). A single-unit query scans 1/47th of the data = 47× cheaper per query.
Backup cost EBS snapshots recommended: +$0.05/GB-month on top of primary storage. At month 120 with backup: +$17,700/month (~$53K/month total). S3 has built-in 11× redundancy (99.999999999% durability). No backup tier needed. Cross-region replication optional: +$0.023/GB.

8 · Notes and Caveats

  • InfluxDB compression varies widely. Simple integer/boolean values compress better than floats; irregular timestamps compress worse. The 8× estimate is typical for mixed IoT time series. Real-world range: 5–15×. Even at 15×, InfluxDB stored/day would be 66 GB (similar to Parquet), but at $0.08/GB vs $0.023/GB — EBS vs S3 alone is 3.5× more expensive.
  • InfluxDB Cloud pricing differs. InfluxDB Cloud (IOx engine) uses Parquet on S3 internally and charges ~$0.20/GB-month for storage — roughly 8.7× more than raw S3. Cloud plans also meter writes by data point count, adding $0.002–0.006/MB written on top of storage.
  • Athena cost is query-dependent. The $150/month estimate assumes typical monitoring queries scanning ~30 TB/month. With aggressive partitioning (topic_partition_segments: 2), per-unit queries scan 1/47 of total data; Athena costs could drop to $10–50/month. Heavy analytics could push costs higher.
  • InfluxDB downsampling is common in practice. Operators often configure continuous queries to aggregate data older than 30 days (e.g., 1-minute intervals). This reduces long-term storage but loses the original resolution — a trade-off that Parquet avoids entirely: all raw rows are preserved cheaply.
  • EC2 for InfluxDB may need to be larger. At 81,420 writes/sec sustained, r5.2xlarge is a lower bound. Production deployments often use r5.4xlarge ($730/month) or run a two-node cluster for HA ($1,460/month), increasing the 10-year total by $175K–$1.05M beyond the figures shown here.
  • Parquet writer runs on the edge. writer.cpp runs on the same machine as FlashMQ (the edge host). No additional EC2 is needed for data collection. The only AWS EC2 in the Parquet path is the small DuckDB query server.
  • Fleet scale multiplier. These figures are for a single site (47 BMS units). At 10 sites the costs scale proportionally; at 100 sites the 10-year InfluxDB bill approaches $218M vs $36M for Parquet/S3 Standard.
