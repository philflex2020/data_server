EMS Physics Generator — Proposal

Replace uniform-random signal pools with a physics-based state machine that produces believable BMS/PCS charge-discharge behaviour.

Ground Truth — SBESS3 Real Site Data

SBESS3 is a utility-scale Battery Energy Storage System (US site, 60 Hz grid). InfluxDB export covers a 3-hour operational window (2026-03-26 19:26–22:26 UTC) across two unit controllers (UC 61B5, UC 63A1). 8 CSV files, 2.7 GB uncompressed, ~2.5M rows of signal data. This window shows the system in active discharge (peak-shaving).

What's Wrong Today

Every signal is a uniform random number 0–100 every sweep. The current ems_site_simulator pre-fills pools of random floats and cycles through them. It is fast (~81k msg/s) but produces data that no real system could ever generate.

This matters because: compression ratios are unrealistic (random data doesn't compress), query patterns are meaningless, and the data cannot be used to demonstrate anomaly detection or alerting.

Charge / Discharge Cycle Model

The simulation accelerates this cycle so a configurable sweep count covers the full 24-hour profile. Each unit is offset in the cycle by a small random phase so units are not all at the same SOC simultaneously.

Proposed Implementation

New file: source/stress_runner/ems_physics_runner.py. Replaces the random-pool approach in ems_site_simulator (used by the compare benchmark and gx10 4-sim stack) while keeping the same MQTT topic format and publish rate.

Key design choice: Keep the publish rate unchanged (~1800 msg/s per unit). The physics engine updates state once per sweep (all signals for one unit in one pass), then publishes each signal as a separate MQTT message as today. No changes to the writer configs.

State per unit

class UnitPhysics:
    # Cycle state machine
    state       : str          # IDLE | CHARGING | DISCHARGING | FAULT
    state_elapsed: float       # seconds in current state
    cycle_phase : float        # 0.0–1.0, initial offset per unit

    # Integrator
    soc         : float        # 0–100 %
    soh         : float        # 94–100 %, constant per unit

    # Thermals (per cell, RC model)
    cell_temp   : list[float]  # one per cell, °C
    cell_imbalance: list[float]# fixed ±2 mV offset per cell

    # Grid / PCS
    grid_freq   : float        # Hz, slow random walk

    # Rare event timers
    bms_fault_ttl    : float   # seconds remaining, 0 = clear
    bms_warning_ttl  : float
    pcs_fault_ttl    : float
    door_open_ttl    : float

State machine transitions

# Simulated day = 600 sweeps (configurable via --day-sweeps N)
# Default cycle (fraction of day):
CYCLE = [
    ("IDLE",        0.25),   # 00:00–06:00  rest at ~40% SOC
    ("CHARGING",    0.25),   # 06:00–12:00  constant-current charge to 95%
    ("REST",        0.08),   # 12:00–14:00  post-charge rest
    ("DISCHARGING", 0.33),   # 14:00–22:00  peak-shaving discharge
    ("IDLE",        0.09),   # 22:00–24:00  night rest
]

# SOC → open-circuit cell voltage (V), Li-NMC typical

Rare event scheduling (Poisson)

def maybe_trigger_fault(unit, dt_sim, rate_per_day):
    # rate_per_day = average faults per simulated day
    # dt_sim = fraction of day this sweep represents
    prob = 1 - math.exp(-rate_per_day * dt_sim)
    if random.random() < prob:
        # trigger fault: pick code, set TTL
        unit.bms_fault_code = random.randint(1, 15)
        unit.bms_fault_ttl  = random.uniform(60, 600)  # sim seconds
