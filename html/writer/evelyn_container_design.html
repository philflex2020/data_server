1. Overview

Option C bundles all stack components into a single Docker image. Deployment on any Linux box is a single docker run command. No systemd knowledge required. No docker-compose orchestration. No separate service installation.

ComponentRoleManaged by
FlashMQMQTT broker — accepts EMS hardware connections + bridgesupervisord
parquet_writerMQTT subscriber → Parquet writersupervisord
bridge_api_container.pyREST management API — config, storage, bridge, /query/currentsupervisord
nginxServes baked-in HTML dashboard pagessupervisord

External only: the Parquet data volume (/data/parquet) and the config volume (/etc/evelyn). Everything else lives inside the container — including both C++ binaries, which are compiled from source during docker build. No pre-built binaries required on the host.

The key architectural difference from Options A and B: bridge_api manages processes via supervisorctl instead of docker compose or systemctl. No Docker socket mount required. No systemd on the host required.

3. Dockerfile

Three-stage build. Stage 1 compiles FlashMQ from source. Stage 2 compiles parquet_writer from source. Stage 3 assembles the runtime image — no pre-built binaries required on the host.

docker/evelyn-container/Dockerfile
# ── Stage 1: Build FlashMQ ───────────────────────────────────────────────

ENV DEBIAN_FRONTEND=noninteractive

RUN apt-get update && apt-get install -y \
    cmake build-essential git libssl-dev \
    && rm -rf /var/lib/apt/lists/*

RUN git clone --branch v1.9.1 --depth 1 \
    https://github.com/halfgaar/FlashMQ.git /flashmq  # latest as of 2026-03-30

RUN cmake -B /flashmq/build -S /flashmq -DCMAKE_BUILD_TYPE=Release \

    libssl-dev libsimdjson-dev \

4. What is supervisord?

supervisord is a Python process manager — the container equivalent of systemctl. It runs as PID 1 inside the container and is responsible for starting, watching, and restarting all other processes.

Docker containers expect a single process as PID 1. If that process dies, the container exits. But we need four processes running simultaneously. supervisord owns them all.

One-line description: supervisord is like systemctl but for containers — same mental model, no kernel dependency, runs anywhere Python runs.
systemctl (host)supervisord (container)
Runs onLinux host (systemd)Anywhere Python runs
PID 1systemd is PID 1 of the OSsupervisord is PID 1 of the container
Start/stopsystemctl start foosupervisorctl start foo
Restart on crashRestart=always in unit fileautorestart=true in supervisord.conf
Check statussystemctl status foosupervisorctl status
View logsjournalctl -u foosupervisorctl tail foo
Config lives/etc/systemd/system/foo.service/etc/supervisor/supervisord.conf
ImplementationC, kernel-integratedPure Python — pip install supervisor

supervisord is not a compiled binary — it installs via apt or pip alongside Python. Since bridge_api already requires Python, supervisord adds no new runtime dependency to the container.

bridge_api uses supervisorctl to restart services when configs change:

# bridge config changed → restart FlashMQ
subprocess.run(["supervisorctl", "restart", "flashmq"])

# storage path changed → restart parquet_writer
subprocess.run(["supervisorctl", "restart", "parquet_writer"])

No Docker socket mount required. No systemd on the host required. supervisord communicates via a Unix socket inside the container (/var/run/supervisor.sock).

        listen 8080;
        server_name _;

        index index.html;

        # Serve HTML pages
        location / {
            try_files $uri $uri/ =404;
            add_header Cache-Control "no-cache";
        }

        # Serve parquet files with range request support (for DuckDB-WASM)
        location /parquet/ {
            alias /data/parquet/;
            add_header Accept-Ranges bytes;
            add_header Access-Control-Allow-Origin *;
            add_header Access-Control-Allow-Headers "Range";
            add_header Access-Control-Expose-Headers "Content-Range, Content-Length";
        }

        # Optional: proxy /api/ → bridge_api for single-origin convenience
        location /api/ {
            proxy_pass http://127.0.0.1:8772/;
            proxy_set_header Host $host;
            add_header Access-Control-Allow-Origin *;
        }
    }
}

7. bridge_api_container.py

Rewrite of bridge_api_native.py replacing all systemctl calls with supervisorctl. Same HTTP API surface — existing HTML dashboards work without changes.

docker/evelyn-container/bridge_api_container.py
#!/usr/bin/env python3
"""
bridge_api_container.py — stack management API for Option C container.

Same HTTP API as bridge_api_native.py.
Manages processes via supervisorctl instead of systemctl.

Endpoints
---------
GET  /status          supervisorctl status + health summary
POST /start           supervisorctl start flashmq parquet_writer
POST /stop            supervisorctl stop flashmq parquet_writer
GET  /storage         current parquet output path
POST /storage         {"path":"..."} update writer yaml, mkdir, restart parquet_writer
GET  /bridge          current FlashMQ bridge config
POST /bridge          {"address","port","topic"} write conf, restart flashmq
POST /bridge/clear    remove bridge, restart flashmq
GET  /query/current   DuckDB query on current_state.parquet → JSON
"""
import asyncio, json, os, re, subprocess, sys
try:
    import duckdb
    DUCKDB_AVAILABLE = True
except ImportError:
    DUCKDB_AVAILABLE = False

PORT                = 8772
FLASHMQ_CONF        = "/etc/evelyn/flashmq.conf"
WRITER_YAML         = "/etc/evelyn/writer.yaml"
CURRENT_STATE_PATH  = "/data/parquet/current_state.parquet"
DEFAULT_PATH        = "/data/parquet"

CORS = (
    "Access-Control-Allow-Origin: *\r\n"
    "Access-Control-Allow-Methods: GET, POST, OPTIONS\r\n"
    "Access-Control-Allow-Headers: Content-Type\r\n"
)

BASE_CONF = """\
listen {
    port 1883
    protocol mqtt
    tcp_nodelay true
}

listen {
    port 9001
    protocol websockets
}

allow_anonymous true
log_level warning
storage_dir /var/lib/flashmq
client_initial_buffer_size 4194304
max_packet_size 1048576
max_qos_msg_pending_per_client 65535
thread_count 2
"""

BRIDGE_TEMPLATE = """\

bridge {{
    address {address}
    port {port}
    topic {topic}  in  0
    clientid evelyn-bridge
}}
"""

def run(cmd, timeout=35):
    r = subprocess.run(cmd, capture_output=True, text=True, timeout=timeout)
    return r.returncode == 0, (r.stdout + r.stderr).strip()

def supervisorctl(*args):
    return run(["supervisorctl"] + list(args))

def svc_status():
    """Parse 'supervisorctl status' → dict of {name: {state, pid, uptime}}"""
    ok, out = supervisorctl("status")
    result = {}
    for line in out.splitlines():
        parts = line.split()
        if not parts: continue
        name  = parts[0]
        state = parts[1] if len(parts) > 1 else "UNKNOWN"
        extra = " ".join(parts[2:])
        result[name] = {"state": state, "info": extra}
    return result

def respond(writer, status, body_dict):
    body = json.dumps(body_dict).encode()
    writer.write(
        f"HTTP/1.1 {status}\r\n"
        f"Content-Type: application/json\r\n"
        f"{CORS}"
        f"Content-Length: {len(body)}\r\n"
        f"Connection: close\r\n\r\n".encode()
        + body
    )

async def handle_request(reader, writer):
    try:
        head = await asyncio.wait_for(reader.read(8192), timeout=10)
        text = head.decode("utf-8", errors="replace")
        lines = text.split("\r\n")
        method, path, *_ = lines[0].split() if lines[0] else ["GET", "/"]
        body_raw = text.split("\r\n\r\n", 1)[1] if "\r\n\r\n" in text else ""

        if method == "OPTIONS":
            writer.write(f"HTTP/1.1 204 No Content\r\n{CORS}Connection: close\r\n\r\n".encode())

        elif path == "/status" and method == "GET":
            svcs = svc_status()
            respond(writer, "200 OK", {"services": svcs, "ok": all(
                s["state"] == "RUNNING" for s in svcs.values())})

        elif path == "/start" and method == "POST":
            ok1, _ = supervisorctl("start", "flashmq")
            ok2, _ = supervisorctl("start", "parquet_writer")
            respond(writer, "200 OK", {"started": ok1 and ok2})

        elif path == "/stop" and method == "POST":
            supervisorctl("stop", "parquet_writer")
            supervisorctl("stop", "flashmq")
            respond(writer, "200 OK", {"stopped": True})

        elif path == "/storage" and method == "GET":
            import yaml
            with open(WRITER_YAML) as f:
                cfg = yaml.safe_load(f)
            p = cfg.get("output", {}).get("base_path", DEFAULT_PATH)
            respond(writer, "200 OK", {"path": p})

        elif path == "/storage" and method == "POST":
            import yaml
            data = json.loads(body_raw)
            new_path = data["path"]
            os.makedirs(new_path, exist_ok=True)
            with open(WRITER_YAML) as f:
                cfg = yaml.safe_load(f)
            cfg.setdefault("output", {})["base_path"] = new_path
            with open(WRITER_YAML, "w") as f:
                yaml.dump(cfg, f)
            supervisorctl("restart", "parquet_writer")
            respond(writer, "200 OK", {"path": new_path, "restarted": True})

        elif path == "/bridge" and method == "GET":
            with open(FLASHMQ_CONF) as f:
                conf = f.read()
            m = re.search(r"bridge\s*\{([^}]+)\}", conf)
            respond(writer, "200 OK", {"bridge": m.group(1).strip() if m else None})

        elif path == "/bridge" and method == "POST":
            data = json.loads(body_raw)
            new_conf = BASE_CONF + BRIDGE_TEMPLATE.format(**data)
            with open(FLASHMQ_CONF, "w") as f:
                f.write(new_conf)
            supervisorctl("restart", "flashmq")
            respond(writer, "200 OK", {"bridge": data, "restarted": True})

        elif path == "/bridge/clear" and method == "POST":
            with open(FLASHMQ_CONF, "w") as f:
                f.write(BASE_CONF)
            supervisorctl("restart", "flashmq")
            respond(writer, "200 OK", {"bridge": None, "restarted": True})

        elif path == "/query/current" and method == "GET":
            if not DUCKDB_AVAILABLE:
                respond(writer, "503 Service Unavailable", {"error": "duckdb not installed"})
            elif not os.path.exists(CURRENT_STATE_PATH):
                respond(writer, "404 Not Found", {"error": "current_state.parquet not written yet"})
            else:
                con = duckdb.connect()
                res = con.execute(f"SELECT * FROM parquet_scan('{CURRENT_STATE_PATH}')")
                cols = [d[0] for d in res.description]
                rows = [dict(zip(cols, r)) for r in res.fetchall()]
                con.close()
                respond(writer, "200 OK", {"columns": cols, "rows": rows})

        else:
            respond(writer, "404 Not Found", {"error": "not found"})

    except Exception as e:
        try: respond(writer, "500 Internal Server Error", {"error": str(e)})
        except: pass
    finally:
        try: writer.close()
        except: pass

async def main():
    server = await asyncio.start_server(handle_request, "0.0.0.0", PORT)
    print(f"[bridge_api] listening on :{PORT}", flush=True)
    async with server:
        await server.serve_forever()

if __name__ == "__main__":
    asyncio.run(main())

8. entrypoint.sh — Config Bootstrap

On first run, the config volume (/etc/evelyn) is empty. The entrypoint script copies the baked-in defaults there, then hands off to supervisord. On subsequent runs the existing configs are left untouched — edits made via bridge_api persist across container restarts and image upgrades.

docker/evelyn-container/entrypoint.sh
#!/usr/bin/env bash
set -euo pipefail

CONFIG_DIR="/etc/evelyn"
DEFAULTS_DIR="/etc/evelyn-defaults"

echo "[entrypoint] checking config volume at $CONFIG_DIR"

# Copy each default file if it doesn't already exist on the volume
for f in "$DEFAULTS_DIR"/*; do
    dest="$CONFIG_DIR/$(basename "$f")"
    if [[ ! -f "$dest" ]]; then
        echo "[entrypoint] installing default: $(basename "$f")"
        cp "$f" "$dest"
    else
        echo "[entrypoint] keeping existing: $(basename "$f")"
    fi
done

echo "[entrypoint] starting supervisord"
exec /usr/bin/supervisord -c /etc/supervisor/supervisord.conf
First run (empty config volume): copies flashmq.conf and writer.yaml from defaults, starts supervisord.
Subsequent runs: existing configs left in place — bridge changes, storage path edits, and custom bridge configs all survive container restarts and image upgrades.

To reset to defaults (e.g. factory reset a site):

# Remove the config volume and recreate — defaults re-installed on next start
docker stop evelyn && docker rm evelyn
sudo rm -rf /etc/evelyn-site/*
docker run ... (same command as before)

13. Limitations & Open Questions

FlashMQ persistence dir: /var/lib/flashmq is inside the container — ephemeral by default. Retained QoS-1 messages are lost on container restart unless this is also volume-mounted. For QoS-0 only (our current mode) this is acceptable.
Supervisor logs: /var/log/supervisor/ is ephemeral. Use docker logs evelyn (supervisord stdout) or mount a log volume: -v /data/logs/evelyn:/var/log/supervisor. nginx logs are redirected to stdout/stderr so supervisord captures and rotates them — no separate /var/log/nginx/ volume needed.
Config persistence: resolved — /etc/evelyn is an external volume (-v /etc/evelyn-site:/etc/evelyn). bridge_api edits (bridge config, storage path) persist across restarts and image upgrades. Defaults are re-installed only if a config file is missing.
WAL files: WAL lives in /data/parquet/.wal — inside the data volume, so survives container restarts. parquet_writer replays WAL on startup. Crash recovery works correctly.
current_state.parquet: written to /data/parquet/current_state.parquet — inside the volume, persistent. GET /query/current works as soon as parquet_writer has completed its first flush.
