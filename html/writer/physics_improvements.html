Motivation

The original simulator used a Longbow LFP pack OCV curve and a bare-bones peripheral model (7 site signals, 6 meter signals, 7 PCS per-unit signals). Analysis of real Evelyn BESS site parquet data revealed several discrepancies and gaps:

  • OCV curve overshoots real pack voltage by ~47 V on average.
  • Internal resistance is constant; real LFP Rint rises sharply at SOC extremes.
  • No coulombic efficiency factor — charge/discharge energy asymmetry ignored.
  • Rack SOC imbalance is static (±0.1%) — real racks diverge dynamically over cycles.
  • RTAC peripheral entirely absent (348 evelyn signals — feeder meters + QSE interface).
  • Site missing AGC, Group power, RTAC_P, PCS Frequency Response, P Correction signals.
  • Meter missing CLR_NET_LOAD_MW / CLR_NET_MW.
  • PCS per-unit missing Hz and symmetrical-component current signal.
  • peripheral_thread_fn existed in code but was never launched from main().

done   #2 — SOC-dependent internal resistance

File: soc_to_voltage_t()

Real LFP packs show Rint rising steeply at SOC extremes (below ~10% and above ~90%). Old model: fixed 8 mΩ regardless of SOC.

// Old return soc_to_voltage(soc) + sign * I * 0.008f; // New: exponential rise at extremes; at SOC=5%: r_factor≈2.0; at SOC=50%: r_factor≈1.0 float r_factor = 1.0f + 1.5f * expf(-0.12f * fminf(soc, 100.0f - soc)); return soc_to_voltage(soc) + sign * I * 0.008f * r_factor;

done   #3 — Cell count updated to match evelyn pack

File: soc_to_cell_v()

At SOC=94%, evelyn real voltage = 1137 V. With LFP max cell voltage 3.5 V: cells ≈ 1137/3.50 ≈ 325 cells (vs old 450-cell Longbow assumption). Affects undervolt/overvolt fault thresholds.

// Old: 450 cells static float soc_to_cell_v(float soc) { return soc_to_voltage(soc) / 450.0f; } // New: ~325 cells (evelyn pack) static float soc_to_cell_v(float soc) { return soc_to_voltage(soc) / 325.0f; }

done   #4 — Coulombic efficiency in SOC integration

File: physics_thread_fn() SOC update

Real LFP: ~96% charge efficiency, ~98% discharge efficiency. This creates realistic asymmetry in charge_kwh vs discharge_kwh counters.

// Old float dsoc = (I / u.capacity) * (100.0f / 3600.0f) * DT; if (charging) u.soc += dsoc; else u.soc -= dsoc; // New float eta = (u.mode == Mode::CHARGING) ? 0.96f : (1.0f / 0.98f); float dsoc = (I / u.capacity) * (100.0f / 3600.0f) * DT * eta;

done   #5 — Dynamic rack SOC drift

File: physics_thread_fn() rack SOC update

Old model: static ±0.1% spread, never evolves. New: drift accumulates per-rack at ±2×10−4×|dsoc| per step, clamped ±3%.

// Old: static spread float noise = (float)(ri - 2) * 0.05f; u.racks[ri].soc = clamp(u.soc + noise); // New: accumulating drift u.racks[ri].drift += (float)(ri - 2) * 2e-4f * fabsf(dsoc); u.racks[ri].drift = clamp(u.racks[ri].drift, -3.0f, 3.0f); u.racks[ri].soc = clamp(u.soc + u.racks[ri].drift);

Known Limitations / Future Work

IssueStatusNotes
SysCurrent CV=6.7 (extreme noise) noted Real high CV is due to near-zero mean + high-amplitude transients during frequency response and mode transitions — not reproducible with the current uniform noise pool model. Would require mixture distribution or event-driven current injection. Logged for future work.
RTAC: 348 total signals vs ~15 simulated partial Only the signals visible in the noise report are simulated. Full RTAC signal list requires a dedicated RTAC template file.
Power signal CV ~3–5 in real data noted Uniform noise pool gives CV ≈ 0.14 at 50% amplitude. Real variance comes from dispatch setpoint changes (slow) + measurement noise (fast). Without a realistic dispatch profile the slow variation cannot be reproduced.
Self-discharge noted LFP self-discharge ~0.5–2%/month — negligible for short-duration simulations.
