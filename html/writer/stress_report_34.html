Key Findings

Zero data loss at 600k msg/s

All four writers ran for 3 days with zero overflow drops and zero ring drops. Ring buffer (ring_cap: 262144) absorbed burst traffic without loss. Combined: ~160.9 billion rows written cleanly.

~160% CPU per writer at 150k msg/s

Each writer uses ~1.6 cores. Linear scaling — no contention between instances. At Evelyn rate (175–81,420 msg/s), expect <10% per writer.

~750 MB RSS per writer — stable

Memory is flat after 3 days — no growth or leak. Total writer footprint ~3 GB against 256 GB available. 1.2% of RAM.

FlashMQ: 141% CPU / 105 MB at 600k fan-out

Single broker handles 600k msg/s → 4 subscribers efficiently. Scales well — for Evelyn 46-unit load (81k msg/s) this will be trivial.

Flush latency: avg 42–46 ms, max 310–431 ms

10-second flush interval produces files of 3k–50k rows depending on rate. Max spikes are transient disk write latency — no queue backup observed.

Machine at ~21% CPU — headroom for 4× Evelyn writers

Full torture stack (600k msg/s × 4) uses ~12 cores. Evelyn demo at 81k msg/s total will use a small fraction. Compactor threads add negligible overhead.
