Tesla Edge Storage — Telemetry Architecture

Reference notes · mapping to Evelyn edge ingest problem

On-vehicle (edge)

  • Logs to a local SQLite database on the vehicle's Linux filesystem
  • Also writes binary .clog / .bz2 compressed log files per subsystem (CAN bus, motor controller, thermal, etc.)
  • A dedicated "diag" partition (~few GB) is reserved on the eMMC
  • Sampling rates vary by signal:
    • High-frequency (motor current, torque): ~100 Hz
    • Slow signals (SOC, temperatures): ~1 Hz

Upload to Tesla servers

  • Opportunistic — triggered when parked on WiFi (home, Supercharger)
  • Batched, not real-time
  • Estimated ~1–2 GB/day per vehicle of raw telemetry
  • At fleet scale (~6M vehicles): potentially petabytes/day ingested
  • Destination: Tesla backend (mix of AWS and own DCs)

Compression & reduction techniques

Delta encoding
Store first value then only the difference. A signal holding at 72.3 °C for 30 s becomes one value plus a run of near-zero deltas.
Run-length encoding (RLE)
For boolean / discrete signals that hold a value for extended periods, store value + duration rather than repeating samples.
Adaptive downsampling
High-frequency signals logged at full rate only during transients. When signal variance is low the log rate drops to ~1 Hz automatically.
Deadband threshold
A new sample is only written if the value has moved by more than X% from the last logged value — applied at logger level, not downstream.
Protobuf serialisation
Binary wire format (not JSON), schema-versioned. Compact and CPU-efficient to encode/decode.

Key gap in current Evelyn approach

Our Filter B uses dedup_interval = 60s — time-based: one sample passes per minute regardless of whether the value changed.

Tesla's deadband is value-based: only store on meaningful change.

Conservative scenario (~27% reduction) — time-based dedup (current)

Aggressive scenario (~85% reduction) — value-based deadband (stretch goal)

Telegraf native processors.dedup does not support deadband. A starlark processor would be needed to implement per-signal thresholds.
