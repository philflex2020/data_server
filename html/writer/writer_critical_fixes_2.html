Step 2 — Make the signal pipe non-blocking in main() (after pipe() call, ~line 1139):

Add #include <fcntl.h> near the other POSIX includes.

Step 3 — Replace notify_one() calls in on_message with a pipe write (lines 984 and 999): 
        // overflow path (line 982–985):
        g_flush_needed.store(true);  // m2: sticky — survives a busy flush thread
        g_flush_cv.notify_one();
        return;
        // per-partition path (line 997–1000):
        g_flush_needed.store(true);  // m2: sticky
        g_flush_cv.notify_one();

        g_flush_needed.store(true);

The (void) cast suppresses the unused-result warning. Writing to the pipe from on_message (MQTT network thread) and from handle_signal (signal context) is safe — write() on a pipe is async-signal-safe and thread-safe.

Step 4 — Drain the pipe in flush_thread_fn() before each flush cycle to prevent spin (lines 849–861):

    while (!g_shutdown) {
        // m2: skip the poll if a flush was requested while we were busy last cycle
        if (!g_flush_needed.exchange(false)) {
            struct pollfd pfd = { g_signal_pipe[0], POLLIN, 0 };

static void flush_thread_fn() {
    while (!g_shutdown) {
        if (!g_flush_needed.exchange(false)) {
            struct pollfd pfd = { g_signal_pipe[0], POLLIN, 0 };
            poll(&pfd, 1, g_cfg->flush_interval_seconds * 1000);
        }
        // Drain all bytes from the pipe so the next poll() blocks correctly.
        // The read end is O_NONBLOCK (set in main) so this never blocks.
        { char drain[64]; while (::read(g_signal_pipe[0], drain, sizeof(drain)) > 0) {} }
        std::unique_lock<std::mutex> lock(g_mutex);

Why the drain is required: if on_message writes N bytes to the pipe (N early-flush triggers between two flush cycles), those N bytes sit in the kernel pipe buffer. Without draining, poll() returns immediately on every subsequent call (pipe is still readable), causing the flush thread to spin with ~0ms between empty flushes. The O_NONBLOCK flag makes the drain loop terminate at EAGAIN rather than blocking forever on the last read.

Replace with (write to .tmp then rename atomically): 
arrow::Status flush_partition(const std::string& path,
                               const std::vector& rows,
                               const std::string& compression) {
    if (rows.empty()) return arrow::Status::OK();
    ARROW_ASSIGN_OR_RAISE(auto table, rows_to_arrow_table(rows));
    auto props = parquet::WriterProperties::Builder()
        .compression(to_parquet_compression(compression))
        ->build();
    std::string tmp = path + ".tmp";
    ARROW_ASSIGN_OR_RAISE(auto outfile, arrow::io::FileOutputStream::Open(tmp));
    ARROW_RETURN_NOT_OK(parquet::arrow::WriteTable(
        *table, arrow::default_memory_pool(), outfile,
        static_cast(rows.size()), props));
    ARROW_RETURN_NOT_OK(outfile->Close());   // flush kernel buffers before rename

    std::error_code ec;
    fs::rename(tmp, path, ec);
    if (ec) {
        fs::remove(tmp, ec);   // best-effort cleanup of orphaned .tmp
        return arrow::Status::IOError("flush_partition rename failed: " + ec.message());
    }
    return arrow::Status::OK();
}

Unit tests: all 6 flush_partition tests still pass — the temp file is written and renamed before the test reads the output path. Verified.

WAL note: write_wal_ipc already calls out->Close() explicitly (fixed in C2). This fix brings flush_partition to the same standard.

            safe(o["compression"],           cfg.compression,            "output.compression");

            safe(o["max_messages_per_part"], cfg.max_messages_per_part,  "output.max_messages_per_part");

            safe(w["enabled"], cfg.wal_enabled, "wal.enabled");
            safe(w["path"],    cfg.wal_path,    "wal.path");
        }
        if (auto h = y["health"]) {
            safe(h["enabled"], cfg.health_enabled, "health.enabled");
            safe(h["port"],    cfg.health_port,    "health.port");
        }
        if (auto g = y["guard"]) {
            if (g["min_free_gb"]) {
                try {
                    double gb = g["min_free_gb"].as<double>();

                    std::cerr << "[config] guard.min_free_gb: " << e.what() << " — keeping default\n";
                }
            }
        }
    } catch (const YAML::Exception& e) {
        // Only file-level errors reach here (file not found, top-level parse error)
        std::cerr << "[config] " << e.what() << " — using defaults\n";
    }

The outer try/catch still handles file-not-found and top-level YAML parse errors. All field-level type errors are logged individually and do not abort config loading.
