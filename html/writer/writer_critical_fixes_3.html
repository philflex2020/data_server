F3-1 · SIGPIPE — one line, prevents silent process death

CRITICALR3-C1 · Add SIGPIPE ignore + MSG_NOSIGNAL

main() · after signal(SIGTERM, …) · line ~1231

A monitor client that disconnects mid-response causes the kernel to deliver SIGPIPE. Default disposition: terminate. One line in main() immunises the entire process. Additionally pass MSG_NOSIGNAL on each send() in the health endpoint for defence-in-depth (prevents the signal even on non-Linux POSIX systems).

F3-3 · mosquitto_loop_stop — avoid 30 s Docker timeout

MAJORR3-M1 · Change force=false → force=true

main() · line ~1305

With force=false the MQTT network thread will wait through its full reconnect backoff (up to 30 s) before responding to the stop request. Docker's default stop timeout is 10 s, meaning Docker sends SIGKILL before the writer exits cleanly — skipping the final flush. With force=true the network thread is stopped immediately; mosquitto_destroy() cleans up correctly either way.

mosquitto_disconnect(mosq); - mosquitto_loop_stop(mosq, /*force=*/false); + mosquitto_loop_stop(mosq, /*force=*/true); // don't block up to reconnect_max_seconds

Step 2 — Restructure handle_sync_message():

static void handle_sync_message(const std::string& payload) {
    // --- parse phase (no lock held) ---
    thread_local simdjson::dom::parser sj;
    simdjson::dom::element doc;
    if (sj.parse(payload).get(doc) != simdjson::SUCCESS) return;

    std::string_view session_sv;
    int64_t seq = 0, interval_pub = 0, total_pub = 0;
    if (doc["session_id"].get(session_sv)          != simdjson::SUCCESS) return;
    if (doc["seq"].get(seq)                         != simdjson::SUCCESS) return;
    if (doc["interval_published"].get(interval_pub) != simdjson::SUCCESS) return;
    if (doc["total_published"].get(total_pub)       != simdjson::SUCCESS) total_pub = 0;
    std::string session(session_sv);

    // --- counter update phase (lock acquired here, briefly) ---
    std::lock_guard<std::mutex> lock(g_mutex);

    if (session != g_sync_session_id) {
        if (!g_sync_session_id.empty())
            std::cout << "[sync] new session " << session
                      << " (was " << g_sync_session_id << ") — counters reset\n";
        g_sync_session_id     = session;
        g_received_since_sync = 0;
        g_sync_drops_detected = 0;
        return;
    }

    int64_t received = (g_received_since_sync > static_cast<uint64_t>(INT64_MAX))
                       ? INT64_MAX
                       : static_cast<int64_t>(g_received_since_sync);
    int64_t dropped  = interval_pub - received;
    g_received_since_sync = 0;

    if (dropped > 0) {
        g_sync_drops_detected += static_cast<uint64_t>(dropped);
        std::cerr << "[sync] #" << seq
                  << "  DROPS: expected=" << interval_pub
                  << "  received=" << received
                  << "  dropped=" << dropped
                  << "  cumulative=" << g_sync_drops_detected << "\n";
    } else {
        std::cout << "[sync] #" << seq
                  << "  OK  received=" << received
                  << "  expected=" << interval_pub
                  << "  total_published=" << total_pub << "\n";
    }
}

F3-6 · Row: std::map → std::unordered_map (performance)

PERFR3-P2 · Eliminate red-black tree allocations in hot path

struct Row · line ~227

std::unordered_map uses open addressing with no per-element allocation after the initial bucket array. For small maps (5–15 fields) this is 2–4× faster than std::map and eliminates ~4.5M heap allocations/second at 300k msg/s. No interface changes to callers — both containers expose the same operator[], find(), erase(), iterators.

// A single decoded message row: integer metadata + float measurements struct Row { - std::map<std::string, int64_t> ints; - std::map<std::string, double> floats; + std::unordered_map<std::string, int64_t> ints; + std::unordered_map<std::string, double> floats; };

Also update struct SensorKey which uses a nested map for its identity fields:

struct SensorKey { std::string source_type; - std::map<std::string, int64_t> ints; + std::unordered_map<std::string, int64_t> ints; bool operator<(const SensorKey& o) const { ... } };

⚠ Note on SensorKey: SensorKey is used as a std::map key and relies on operator< which iterates the inner map in sorted order. Switching the inner map to unordered_map means the iteration order in operator< is no longer deterministic. Two workarounds:

  • Option A (simple): keep SensorKey::ints as std::map since the current-state map is only updated on flush (cold path), not the hot ingestion path.
  • Option B (correct fix): rewrite operator< to sort keys explicitly before comparing.

Apply Option A now: change only Row::ints and Row::floats, leave SensorKey::ints as std::map.

// Row — hot path, changed to unordered_map: + std::unordered_map<std::string, int64_t> ints; + std::unordered_map<std::string, double> floats; // SensorKey — cold path (current_state), keep std::map for deterministic ordering: std::map<std::string, int64_t> ints; // keep as std::map (used in operator<)

F3-7 · parse_topic: replace std::istringstream with pointer walk

PERFR3-P1 · Zero-allocation topic parser using string_view

parse_topic() · line ~276

Replaces the std::istringstream + std::getline split with a pointer walk over the const char* already sitting in the ring slot. No heap allocation. No string copies. ~3–5× faster on the critical parse path. 

std::optional<TopicInfo> parse_topic(const std::string& topic) {
    TopicInfo info;
    const char* p   = topic.c_str();
    const char* end = p + topic.size();
    bool first = true;

    while (p <= end) {
        const char* slash = static_cast<const char*>(memchr(p, '/', end - p));
        const char* seg_end = slash ? slash : end;
        std::string_view seg(p, static_cast<size_t>(seg_end - p));

        if (first) {
            // source_type — sanitize and validate
            std::string st(seg);
            auto slash_pos = st.find_first_of("/\\");
            if (slash_pos != std::string::npos) st = st.substr(0, slash_pos);
            if (st.empty() || st == "." || st == "..") return std::nullopt;
            info.source_type = std::move(st);
            first = false;
        } else if (!seg.empty()) {
            auto eq = seg.find('=');
            if (eq != std::string_view::npos) {
                std::string kv_key(seg.substr(0, eq));
                std::string_view val_sv = seg.substr(eq + 1);
                // stoi from string_view — avoid string copy
                int kv_val = 0;
                bool neg = (!val_sv.empty() && val_sv[0] == '-');
                size_t i = neg ? 1 : 0;
                bool has_digit = false;
                for (; i < val_sv.size(); ++i) {
                    if (val_sv[i] < '0' || val_sv[i] > '9') { has_digit = false; break; }
                    has_digit = true;
                    kv_val = kv_val * 10 + (val_sv[i] - '0');
                }
                if (!has_digit) { p = slash ? slash + 1 : end + 1; continue; }
                if (neg) kv_val = -kv_val;
                if (info.kv.count(kv_key))
                    std::cerr << "[topic] duplicate key '" << kv_key << "' in: " << topic << "\n";
                info.kv[kv_key] = kv_val;
            } else {
                // per_cell_item field name (last non-kv segment)
                info.field_name = std::string(seg);
            }
        }

        p = slash ? slash + 1 : end + 1;
    }

    if (info.source_type.empty()) return std::nullopt;
    return info;
}

⚠ The existing sanitize_path_segment() helper is still used for source_type extraction above. The kv segments do not need sanitisation since they are integer-valued (non-numeric values are skipped by the parse).
