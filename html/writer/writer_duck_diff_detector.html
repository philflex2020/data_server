Why DuckDB Over Python/pyarrow

pyarrow approach (diff_detector.html)
  • Reads one file at a time
  • Schema only — no value inspection
  • ~60 lines of Python glue code
  • Can't cross-reference partitions
  • Can't detect value drift (wrong timestamps, wrong ranges)
DuckDB approach (this page)
  • GLOB reads all files in one query
  • Schema and value statistics from parquet_metadata()
  • SQL JOINs across both dirs simultaneously
  • Detects row-count mismatches, value range drift, null inflation, wrong-date partitions
  • NC1 (WAL replay wrong date) — detectable immediately
DuckDB 1.5.0 is available via import duckdb in the repo venv. All queries below are verified against /srv/data/parquet/ on phil-dev. Run from repo root: .venv/bin/python3

Q1 — Schema Diff

Detects: columns added, removed, or type-changed between two writer outputs. This is the highest-priority check — schema changes break all downstream DuckDB queries.

Query 
WITH
old_schema AS (
    SELECT DISTINCT name, type, duckdb_type
    FROM   parquet_schema('/tmp/diff_old/**/*.parquet')
    WHERE  type IS NOT NULL             -- exclude root schema row
),
new_schema AS (
    SELECT DISTINCT name, type, duckdb_type
    FROM   parquet_schema('/tmp/diff_new/**/*.parquet')
    WHERE  type IS NOT NULL
)
SELECT
    COALESCE(o.name, n.name)           AS column_name,
    o.type                             AS old_type,
    n.type                             AS new_type,
    CASE
        WHEN o.name IS NULL            THEN 'ADDED    ← downstream queries safe'
        WHEN n.name IS NULL            THEN 'REMOVED  ← BREAKS downstream queries'
        WHEN o.type != n.type          THEN 'TYPE_CHANGED ← BREAKS mixed-file scans'
        ELSE                                'OK'
    END                                AS status
FROM  old_schema o
FULL OUTER JOIN new_schema n ON o.name = n.name
ORDER BY column_name;

Python snippet 
import duckdb, sys

def schema_diff(old_dir, new_dir):
    con = duckdb.connect()
    rows = con.execute(f"""
        WITH o AS (SELECT DISTINCT name, type FROM parquet_schema('{old_dir}/**/*.parquet') WHERE type IS NOT NULL),
             n AS (SELECT DISTINCT name, type FROM parquet_schema('{new_dir}/**/*.parquet') WHERE type IS NOT NULL)
        SELECT COALESCE(o.name, n.name) AS col,
               o.type AS old_t, n.type AS new_t,
               CASE WHEN o.name IS NULL THEN 'ADDED'
                    WHEN n.name IS NULL THEN 'REMOVED'
                    WHEN o.type != n.type THEN 'TYPE_CHANGED'
                    ELSE 'OK' END AS status
        FROM o FULL OUTER JOIN n ON o.name = n.name
        ORDER BY col
    """).fetchall()
    breaking = [r for r in rows if r[3] in ('REMOVED', 'TYPE_CHANGED')]
    for r in rows:
        tag = '✓' if r[3] == 'OK' else ('✗' if r[3] in ('REMOVED','TYPE_CHANGED') else '△')
        print(f"  {tag}  {r[0]:20s}  {str(r[1]):10s}  {str(r[2]):10s}  {r[3]}")
    return len(breaking) == 0

Q2 — Row Count Diff per Partition

Detects: data loss (fewer rows in new), duplication (more rows), wrong partition routing (NC1 — WAL timestamps). Compare both the total and the per-partition breakdown.

Total row count comparison 
SELECT
    (SELECT COUNT(*) FROM read_parquet('/tmp/diff_old/**/*.parquet', union_by_name=true)) AS old_rows,
    (SELECT COUNT(*) FROM read_parquet('/tmp/diff_new/**/*.parquet', union_by_name=true)) AS new_rows,
    old_rows - new_rows                                                                          AS delta;
Per-partition (site_id) breakdown — immediately reveals NC1 WAL wrong-date routing 
WITH
old_parts AS (
    SELECT site_id, COUNT(*) AS n
    FROM   read_parquet('/tmp/diff_old/**/*.parquet', union_by_name=true)
    GROUP BY site_id
),
new_parts AS (
    SELECT site_id, COUNT(*) AS n
    FROM   read_parquet('/tmp/diff_new/**/*.parquet', union_by_name=true)
    GROUP BY site_id
)
SELECT
    COALESCE(o.site_id, n.site_id)   AS site_id,
    COALESCE(o.n, 0)                 AS old_count,
    COALESCE(n.n, 0)                 AS new_count,
    COALESCE(n.n, 0) - COALESCE(o.n, 0) AS delta,
    CASE
        WHEN o.site_id IS NULL         THEN 'NEW PARTITION'
        WHEN n.site_id IS NULL         THEN 'MISSING PARTITION'
        WHEN n.n < o.n * 0.99          THEN 'DATA LOSS >1%'
        WHEN n.n > o.n * 1.01          THEN 'DUPLICATION >1%'
        ELSE                                 'OK'
    END                               AS status
FROM  old_parts o
FULL OUTER JOIN new_parts n ON o.site_id = n.site_id
ORDER BY site_id;
Detecting NC1 — WAL replay wrote rows to today's date instead of original date: 
-- If old writer had a crash+replay on 2026-03-19, WAL rows for that day
-- go to 2026-03-21 in the new (buggy) writer.  This query surfaces it:
WITH
old_daily AS (
    SELECT
        strftime(
            to_timestamp(MIN(timestamp)),
            '%Y-%m-%d'
        ) AS day,
        COUNT(*) AS n
    FROM read_parquet('/tmp/diff_old/**/*.parquet', union_by_name=true)
    GROUP BY epoch_ms(to_timestamp(timestamp) :: TIMESTAMPTZ :: BIGINT / 86400000)
),
new_daily AS (
    SELECT
        strftime(to_timestamp(MIN(timestamp)), '%Y-%m-%d') AS day,
        COUNT(*) AS n
    FROM read_parquet('/tmp/diff_new/**/*.parquet', union_by_name=true)
    GROUP BY epoch_ms(to_timestamp(timestamp) :: TIMESTAMPTZ :: BIGINT / 86400000)
)
SELECT
    COALESCE(o.day, n.day)                 AS day,
    COALESCE(o.n, 0)                      AS old_row_count,
    COALESCE(n.n, 0)                      AS new_row_count,
    COALESCE(n.n, 0) - COALESCE(o.n, 0)  AS delta
FROM  old_daily o
FULL OUTER JOIN new_daily n ON o.day = n.day
ORDER BY day;
-- NC1 signature: old has rows on 2026-03-19, new has extra rows on 2026-03-21 instead.

Q3 — Column Value Statistics Diff

Uses parquet_metadata() to read per-column min/max/nullcount from Parquet row-group statistics without scanning every row. Extremely fast — reads only file footers.

Column statistics comparison (footer-only read — no row scan) 
WITH
old_stats AS (
    SELECT
        path_in_schema                            AS col,
        MIN(stats_min_value)                     AS min_val,
        MAX(stats_max_value)                     AS max_val,
        SUM(stats_null_count)                    AS nulls,
        SUM(num_values)                          AS total_vals,
        MAX(compression)                         AS codec
    FROM parquet_metadata('/tmp/diff_old/**/*.parquet')
    GROUP BY col
),
new_stats AS (
    SELECT
        path_in_schema AS col,
        MIN(stats_min_value)  AS min_val,
        MAX(stats_max_value)  AS max_val,
        SUM(stats_null_count) AS nulls,
        SUM(num_values)       AS total_vals,
        MAX(compression)      AS codec
    FROM parquet_metadata('/tmp/diff_new/**/*.parquet')
    GROUP BY col
)
SELECT
    o.col,
    o.min_val   AS old_min,   n.min_val  AS new_min,
    o.max_val   AS old_max,   n.max_val  AS new_max,
    o.nulls     AS old_nulls, n.nulls    AS new_nulls,
    o.total_vals AS old_n,    n.total_vals AS new_n,
    o.codec     AS old_codec, n.codec    AS new_codec,
    CASE
        WHEN o.nulls != n.nulls             THEN 'NULL COUNT CHANGED'
        WHEN o.codec != n.codec             THEN 'CODEC CHANGED'
        WHEN o.total_vals != n.total_vals   THEN 'ROW COUNT MISMATCH'
        ELSE                                     'OK'
    END AS flag
FROM  old_stats o
FULL OUTER JOIN new_stats n ON o.col = n.col
ORDER BY o.col;

Null inflation detection — catches NC2 (partial write leaves nulls): 
-- Columns with more nulls in new than old = data was lost mid-write
SELECT
    COALESCE(o.col, n.col)                  AS column_name,
    COALESCE(o.nulls, 0)                  AS old_nulls,
    COALESCE(n.nulls, 0)                  AS new_nulls,
    COALESCE(n.nulls, 0) - COALESCE(o.nulls, 0) AS null_increase
FROM
    (SELECT path_in_schema AS col, SUM(stats_null_count) AS nulls
     FROM parquet_metadata('/tmp/diff_old/**/*.parquet') GROUP BY col) o
FULL OUTER JOIN
    (SELECT path_in_schema AS col, SUM(stats_null_count) AS nulls
     FROM parquet_metadata('/tmp/diff_new/**/*.parquet') GROUP BY col) n
ON o.col = n.col
WHERE COALESCE(n.nulls, 0) > COALESCE(o.nulls, 0)
ORDER BY null_increase DESC;

Q4 — Timestamp Range Diff

The strongest test for NC1 (WAL replay wrong-date routing). If the writer replayed a WAL from last week but used today's timestamp, the rows appear in today's files with stale timestamps — a value-range anomaly that's invisible without this query.

Detect stale timestamps in today's partition (run against a single dir) 
-- Find files where the data timestamps are older than the file's partition date.
-- Partition date is extracted from the directory path.
SELECT
    regexp_extract(file_name, '/(\d{4}/\d{2}/\d{2})/', 1)       AS partition_date,
    MIN(stats_min_value)                                           AS data_ts_min,
    MAX(stats_max_value)                                           AS data_ts_max,
    -- Convert Unix seconds to date string for comparison
    strftime(to_timestamp(CAST(MIN(stats_min_value) AS DOUBLE)), '%Y/%m/%d') AS data_date_min,
    COUNT(DISTINCT file_name)                                      AS file_count,
    SUM(num_values)                                                AS row_count
FROM   parquet_metadata('/tmp/diff_new/**/*.parquet')
WHERE  path_in_schema = 'timestamp'
GROUP BY partition_date
ORDER BY partition_date;
-- NC1 signature: partition_date = '2026/03/21' but data_date_min = '2026/03/19'
--                → WAL from 2026-03-19 was replayed into today's partition.
Timestamp range side-by-side 
SELECT
    'old'                                                         AS src,
    strftime(to_timestamp(MIN(timestamp)), '%Y-%m-%d %H:%M:%S') AS ts_min,
    strftime(to_timestamp(MAX(timestamp)), '%Y-%m-%d %H:%M:%S') AS ts_max,
    MAX(timestamp) - MIN(timestamp)                              AS span_seconds,
    COUNT(*)                                                      AS row_count
FROM read_parquet('/tmp/diff_old/**/*.parquet', union_by_name=true)
UNION ALL
SELECT 'new',
    strftime(to_timestamp(MIN(timestamp)), '%Y-%m-%d %H:%M:%S'),
    strftime(to_timestamp(MAX(timestamp)), '%Y-%m-%d %H:%M:%S'),
    MAX(timestamp) - MIN(timestamp),
    COUNT(*)
FROM read_parquet('/tmp/diff_new/**/*.parquet', union_by_name=true);

Q5 — Partition Directory Structure Diff

Detects changes to the partition path template (e.g., site={site_id} → site_id={site_id}). DuckDB stores the full filename in metadata — we can parse the path structure directly.

File count and size by top-level partition path 
WITH
old_tree AS (
    SELECT
        regexp_replace(file_name, '^.*/(\w+/project=\d+/[^/]+).*$', '\1') AS partition_prefix,
        COUNT(DISTINCT file_name)     AS file_count,
        SUM(total_compressed_size)    AS bytes_compressed,
        SUM(row_group_num_rows)       AS row_count
    FROM parquet_metadata('/tmp/diff_old/**/*.parquet')
    GROUP BY partition_prefix
),
new_tree AS (
    SELECT
        regexp_replace(file_name, '^.*/(\w+/project=\d+/[^/]+).*$', '\1') AS partition_prefix,
        COUNT(DISTINCT file_name) AS file_count,
        SUM(total_compressed_size) AS bytes_compressed,
        SUM(row_group_num_rows)    AS row_count
    FROM parquet_metadata('/tmp/diff_new/**/*.parquet')
    GROUP BY partition_prefix
)
SELECT
    COALESCE(o.partition_prefix, n.partition_prefix)    AS partition,
    COALESCE(o.file_count,  0)                         AS old_files,
    COALESCE(n.file_count,  0)                         AS new_files,
    ROUND(COALESCE(o.bytes_compressed, 0) / 1e6, 1)   AS old_mb,
    ROUND(COALESCE(n.bytes_compressed, 0) / 1e6, 1)   AS new_mb,
    CASE
        WHEN o.partition_prefix IS NULL  THEN 'NEW PARTITION'
        WHEN n.partition_prefix IS NULL  THEN 'MISSING PARTITION'
        WHEN o.row_count != n.row_count   THEN 'ROW COUNT DIFFERS'
        ELSE                                   'OK'
    END  AS status
FROM  old_tree o
FULL OUTER JOIN new_tree n ON o.partition_prefix = n.partition_prefix
ORDER BY partition;

Q6 — Compression and File Size Diff

Catches codec changes (snappy → zstd etc.) and detects significantly larger/smaller files that might indicate schema changes or data distribution changes.

Compression ratio and codec per column 
SELECT
    path_in_schema                                       AS col,
    SUM(total_uncompressed_size)                         AS uncompressed_bytes,
    SUM(total_compressed_size)                           AS compressed_bytes,
    ROUND(SUM(total_uncompressed_size) * 1.0 /
          NULLIF(SUM(total_compressed_size), 0), 2)  AS compression_ratio,
    MAX(compression)                                     AS codec
FROM   parquet_metadata('/srv/data/parquet/**/*.parquet')
GROUP BY col
ORDER BY uncompressed_bytes DESC;

Q8 — Hysteresis: Noise Suppression and Frozen Sensor Detection

Two separate problems, one tool. Noisy sensors write nearly identical values on every message — flooding storage with rows that carry no new information. Frozen sensors never change — indicating a dead sensor, stuck ADC, or lost upstream feed. Both are invisible to the schema and row-count checks above. These queries work on a single dataset (no second writer needed).

Step 1 — Suggest a hysteresis threshold from the data

The right hyst value sits between the sensor's natural noise floor and its smallest meaningful step change. This query computes the distribution of consecutive value deltas per sensor — a much better noise estimate than overall stddev, which conflates real drift with measurement noise. 

-- Consecutive-delta stddev per sensor → natural noise floor
-- Run on a sample window (e.g. last hour) to keep it fast
WITH
readings AS (
    SELECT
        site_id, rack_id, module_id, cell_id,
        timestamp, value,
        value - LAG(value) OVER (
            PARTITION BY site_id, rack_id, module_id, cell_id
            ORDER BY timestamp
        ) AS delta
    FROM read_parquet('/srv/data/parquet/**/*.parquet', union_by_name=true)
    WHERE timestamp > (SELECT MAX(timestamp) - 3600
                        FROM read_parquet('/srv/data/parquet/**/*.parquet', union_by_name=true))
),
per_sensor AS (
    SELECT
        site_id, rack_id, module_id, cell_id,
        COUNT(*)                         AS n_readings,
        ROUND(STDDEV(delta),      6)   AS delta_stddev,      -- noise floor (1σ)
        ROUND(STDDEV(delta) * 3, 6)   AS suggested_hyst,    -- 3σ dead-band
        ROUND(MIN(value),  4)          AS val_min,
        ROUND(MAX(value),  4)          AS val_max,
        ROUND(MAX(value) - MIN(value), 4) AS val_range
    FROM readings
    WHERE delta IS NOT NULL
    GROUP BY site_id, rack_id, module_id, cell_id
)
SELECT *,
    CASE
        WHEN val_range = 0                THEN 'FROZEN — value never changed'
        WHEN delta_stddev > val_range * 0.1 THEN 'HIGH NOISE — delta stddev > 10% of range'
        WHEN delta_stddev > val_range * 0.01THEN 'NOISY — delta stddev > 1% of range'
        ELSE                                    'OK'
    END AS sensor_status
FROM per_sensor
ORDER BY delta_stddev DESC;
Expected output on production data (synthetic random payload — all sensors will show high noise by design):

-------  -------  ---------  -------  ----------  ------------  --------------  --------  --------  ---------  -------------
     10        0          0        0        4321        57.8821        173.646   -41.65   104.99     146.64    HIGH NOISE — delta stddev > 10% of range
     10        0          0        1        4321        57.9103        173.731   -41.65   104.99     146.64    HIGH NOISE — delta stddev > 10% of range

With real battery cells: delta_stddev of a stable cell at rest is typically 0.001–0.01 V (ADC quantization noise). A suggested_hyst of 3σ ≈ 0.003–0.03 V gives a dead-band that ignores ADC jitter while passing genuine charge/discharge transitions (which produce step changes of 0.1–1 V/sec).

Step 2 — Fleet-wide hyst summary (one recommended value)

Aggregate across all sensors to get a single value you can drop into writer_config.yaml:

WITH
deltas AS (
    SELECT
        value - LAG(value) OVER (
            PARTITION BY site_id, rack_id, module_id, cell_id
            ORDER BY timestamp
        ) AS d
    FROM read_parquet('/srv/data/parquet/**/*.parquet', union_by_name=true)
    WHERE timestamp > (SELECT MAX(timestamp) - 3600
                        FROM read_parquet('/srv/data/parquet/**/*.parquet', union_by_name=true))
)
SELECT
    ROUND(PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY ABS(d)), 6) AS median_abs_delta,
    ROUND(PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY ABS(d)), 6) AS p95_abs_delta,
    ROUND(PERCENTILE_CONT(0.99) WITHIN GROUP (ORDER BY ABS(d)), 6) AS p99_abs_delta,
    -- Conservative: ignore changes below median → halves write volume for stable cells
    ROUND(PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY ABS(d)), 6) AS hyst_conservative,
    -- Aggressive: only write top-5% changes → drops ~95% of writes
    ROUND(PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY ABS(d)), 6) AS hyst_aggressive
FROM deltas
WHERE d IS NOT NULL;

Step 3 — Detect frozen sensors

Sensors where the value did not change at all over the observation window — hardware fault, disconnected cell, or upstream silence.

SELECT
    site_id, rack_id, module_id, cell_id,
    COUNT(*)                           AS n_readings,
    MIN(value)                         AS value_always,
    MIN(to_timestamp(timestamp))       AS first_seen,
    MAX(to_timestamp(timestamp))       AS last_seen,
    ROUND(MAX(timestamp) - MIN(timestamp), 0) AS window_seconds
FROM read_parquet('/srv/data/parquet/**/*.parquet', union_by_name=true)
GROUP BY site_id, rack_id, module_id, cell_id
HAVING MAX(value) = MIN(value)       -- exact match: value literally never changed
ORDER BY n_readings DESC;

For near-frozen sensors (tiny jitter but effectively stuck), use HAVING STDDEV(value) < hyst instead of exact equality.

Step 4 — Detect high-frequency no-change floods

Sensors that change every single reading add no incremental information but consume storage and inflate row counts. This identifies sensors where fewer than 5% of readings differ from the previous by more than the proposed hyst — good candidates for dead-band filtering at the writer. 

WITH
tagged AS (
    SELECT
        site_id, rack_id, module_id, cell_id, value, timestamp,
        LAG(value) OVER (
            PARTITION BY site_id, rack_id, module_id, cell_id
            ORDER BY timestamp
        ) AS prev_value
    FROM read_parquet('/srv/data/parquet/**/*.parquet', union_by_name=true)
    WHERE timestamp > (SELECT MAX(timestamp) - 3600
                        FROM read_parquet('/srv/data/parquet/**/*.parquet', union_by_name=true))
)
SELECT
    site_id, rack_id, module_id, cell_id,
    COUNT(*)                                             AS n,
    ROUND(AVG(CASE WHEN ABS(value - prev_value) < 0.05  -- substitute your hyst here
                   THEN 1.0 ELSE 0.0 END) * 100, 1)     AS pct_below_hyst,
    ROUND(STDDEV(value - prev_value), 6)               AS delta_stddev
FROM tagged
WHERE prev_value IS NOT NULL
GROUP BY site_id, rack_id, module_id, cell_id
HAVING  AVG(CASE WHEN ABS(value - prev_value) < 0.05 THEN 1.0 ELSE 0.0 END) > 0.95
ORDER BY pct_below_hyst DESC;

How hyst maps to writer.cpp

The queries above tell you what hyst to use. The writer enforces it via the g_current_state map (already maintained for the current-state parquet file). The proposed config key and on_message gate: 

# writer_config.yaml — proposed
output:
  value_hysteresis: 0.05    # skip row if |new_value - last_value| < this; 0 = off
// on_message — proposed dead-band gate (after current-state lookup)
if (g_cfg->value_hysteresis > 0.0) {
    auto it = g_current_state->find(sensor_key);
    if (it != g_current_state->end()) {
        auto prev_it = it->second.floats.find("value");
        auto new_it  = row.floats.find("value");
        if (prev_it != it->second.floats.end() && new_it != row.floats.end())
            if (std::abs(new_it->second - prev_it->second) < g_cfg->value_hysteresis)
                return;   // below dead-band — discard, do not buffer
    }
}
Set hyst conservatively — a value too high silently discards real state changes. Use the Step 2 fleet-wide p95 as a starting point; monitor overflow_drops and row rate via /health to confirm the reduction is proportional to what the query predicted.

Q7 — WAL File Schema Diff

WAL files are Arrow IPC format, not Parquet — DuckDB can't read them directly. Use pyarrow to convert the schema to a temp Parquet file for comparison, or read metadata via Python.

Python: compare WAL schema between two WAL directories 
import pyarrow.ipc as ipc, pyarrow as pa, pathlib, duckdb

def wal_schemas(wal_dir: str) -> dict:
    """Return {filename: schema} for all .wal.arrow files in dir."""
    schemas = {}
    for f in pathlib.Path(wal_dir).glob("*.wal.arrow"):
        with pa.memory_map(str(f), 'r') as source:
            reader = ipc.open_file(source)
            schemas[f.name] = reader.schema_arrow
    return schemas

def wal_diff(old_wal_dir: str, new_wal_dir: str):
    old = wal_schemas(old_wal_dir)
    new = wal_schemas(new_wal_dir)

    old_fields = {f.name: str(f.type) for s in old.values() for f in s}
    new_fields = {f.name: str(f.type) for s in new.values() for f in s}

    all_cols = sorted(set(old_fields) | set(new_fields))
    breaking = False
    for col in all_cols:
        ot, nt = old_fields.get(col), new_fields.get(col)
        if ot is None:   print(f"  △ ADDED    {col}: {nt}")
        elif nt is None: print(f"  ✗ REMOVED  {col}: {ot}"); breaking = True
        elif ot != nt:   print(f"  ✗ CHANGED  {col}: {ot} → {nt}"); breaking = True
        else:            print(f"  ✓ OK       {col}: {ot}")
    return not breaking

Full Automated Diff Script

Save as scripts/duck_diff.py. Takes two directories, runs all checks, exits non-zero if any BREAK-severity issue is found.

#!/usr/bin/env python3
"""
duck_diff.py  —  DuckDB-based diff detector for parquet_writer outputs

Usage:
    python3 scripts/duck_diff.py OLD_DIR NEW_DIR [--verbose]

Exit codes:
    0  all checks pass
    1  BREAK-severity finding (column removed, type changed, row count mismatch)
    2  WARN-severity finding only (column added, codec changed)
"""
import sys, argparse, duckdb

def run(con, sql, label):
    try:
        return con.execute(sql).fetchall()
    except Exception as e:
        print(f"  [ERROR] {label}: {e}")
        return []

def main():
    ap = argparse.ArgumentParser()
    ap.add_argument("old_dir")
    ap.add_argument("new_dir")
    ap.add_argument("--verbose", action="store_true")
    args = ap.parse_args()

    old = args.old_dir.rstrip("/")
    new = args.new_dir.rstrip("/")
    con = duckdb.connect()

    worst = 0   # 0=OK, 1=WARN, 2=BREAK

    # ── Q1: Schema diff ───────────────────────────────────────────────────
    print("\n[1/5] Schema diff")
    rows = run(con, f"""
        WITH o AS (SELECT DISTINCT name, type FROM parquet_schema('{old}/**/*.parquet') WHERE type IS NOT NULL),
             n AS (SELECT DISTINCT name, type FROM parquet_schema('{new}/**/*.parquet') WHERE type IS NOT NULL)
        SELECT COALESCE(o.name, n.name),
               o.type, n.type,
               CASE WHEN o.name IS NULL THEN 'ADDED'
                    WHEN n.name IS NULL THEN 'REMOVED'
                    WHEN o.type != n.type THEN 'TYPE_CHANGED'
                    ELSE 'OK' END
        FROM o FULL OUTER JOIN n ON o.name = n.name ORDER BY 1
    """, "schema_diff")
    for col, ot, nt, status in rows:
        sym = "✓" if status == "OK" else ("△" if status == "ADDED" else "✗")
        print(f"  {sym}  {col:20s}  {str(ot):10s}  →  {str(nt):10s}  {status}")
        if status in ("REMOVED", "TYPE_CHANGED"): worst = max(worst, 2)
        if status == "ADDED":                     worst = max(worst, 1)

    # ── Q2: Row count diff ────────────────────────────────────────────────
    print("\n[2/5] Row count diff")
    rows = run(con, f"""
        WITH o AS (SELECT site_id, COUNT(*) AS n FROM read_parquet('{old}/**/*.parquet', union_by_name=true) GROUP BY site_id),
             n AS (SELECT site_id, COUNT(*) AS n FROM read_parquet('{new}/**/*.parquet', union_by_name=true) GROUP BY site_id)
        SELECT COALESCE(o.site_id, n.site_id),
               COALESCE(o.n, 0), COALESCE(n.n, 0),
               COALESCE(n.n,0) - COALESCE(o.n,0),
               CASE WHEN o.site_id IS NULL THEN 'NEW_PARTITION'
                    WHEN n.site_id IS NULL THEN 'MISSING_PARTITION'
                    WHEN COALESCE(n.n,0) < COALESCE(o.n,0)*0.99 THEN 'DATA_LOSS'
                    WHEN COALESCE(n.n,0) > COALESCE(o.n,0)*1.01 THEN 'DUPLICATION'
                    ELSE 'OK' END
        FROM o FULL OUTER JOIN n ON o.site_id = n.site_id ORDER BY 1
    """, "row_count_diff")
    for sid, on, nn, delta, status in rows:
        sym = "✓" if status == "OK" else ("△" if "NEW" in status else "✗")
        print(f"  {sym}  site={str(sid):4s}  old={on:>10,}  new={nn:>10,}  delta={delta:+,}  {status}")
        if status in ("MISSING_PARTITION", "DATA_LOSS"): worst = max(worst, 2)
        if status in ("NEW_PARTITION", "DUPLICATION"):   worst = max(worst, 1)

    # ── Q3: Null count diff ───────────────────────────────────────────────
    print("\n[3/5] Null count diff (footer-only read)")
    rows = run(con, f"""
        WITH o AS (SELECT path_in_schema AS col, SUM(stats_null_count) AS nulls
                   FROM parquet_metadata('{old}/**/*.parquet') GROUP BY col),
             n AS (SELECT path_in_schema AS col, SUM(stats_null_count) AS nulls
                   FROM parquet_metadata('{new}/**/*.parquet') GROUP BY col)
        SELECT COALESCE(o.col, n.col),
               COALESCE(o.nulls,0), COALESCE(n.nulls,0),
               COALESCE(n.nulls,0) - COALESCE(o.nulls,0)
        FROM o FULL OUTER JOIN n ON o.col = n.col
        ORDER BY 1
    """, "null_diff")
    for col, on, nn, delta in rows:
        sym = "✓" if delta == 0 else "✗"
        print(f"  {sym}  {col:20s}  old_nulls={on:>8,}  new_nulls={nn:>8,}  delta={delta:+,}")
        if delta > 0: worst = max(worst, 2)

    # ── Q4: Timestamp range ───────────────────────────────────────────────
    print("\n[4/5] Timestamp range (detects NC1 WAL wrong-date routing)")
    rows = run(con, f"""
        SELECT 'old' AS src,
               strftime(to_timestamp(MIN(timestamp)), '%Y-%m-%d %H:%M') AS ts_min,
               strftime(to_timestamp(MAX(timestamp)), '%Y-%m-%d %H:%M') AS ts_max,
               COUNT(*) AS n
        FROM read_parquet('{old}/**/*.parquet', union_by_name=true)
        UNION ALL
        SELECT 'new',
               strftime(to_timestamp(MIN(timestamp)), '%Y-%m-%d %H:%M'),
               strftime(to_timestamp(MAX(timestamp)), '%Y-%m-%d %H:%M'),
               COUNT(*)
        FROM read_parquet('{new}/**/*.parquet', union_by_name=true)
    """, "timestamp_range")
    for src, ts_min, ts_max, n in rows:
        print(f"  {src:4s}  rows={n:>12,}  ts_min={ts_min}  ts_max={ts_max}")

    # ── Q5: Codec diff ────────────────────────────────────────────────────
    print("\n[5/5] Compression codec diff")
    rows = run(con, f"""
        WITH o AS (SELECT path_in_schema AS col, MAX(compression) AS codec
                   FROM parquet_metadata('{old}/**/*.parquet') GROUP BY col),
             n AS (SELECT path_in_schema AS col, MAX(compression) AS codec
                   FROM parquet_metadata('{new}/**/*.parquet') GROUP BY col)
        SELECT COALESCE(o.col, n.col), o.codec, n.codec
        FROM o FULL OUTER JOIN n ON o.col = n.col
        WHERE o.codec IS DISTINCT FROM n.codec
        ORDER BY 1
    """, "codec_diff")
    if rows:
        for col, oc, nc in rows:
            print(f"  △  {col:20s}  {oc} → {nc}")
            worst = max(worst, 1)
    else:
        print("  ✓  No codec changes")

    # ── Summary ───────────────────────────────────────────────────────────
    print()
    if worst == 0:   print("✓ ALL CHECKS PASSED — outputs are equivalent")
    elif worst == 1: print("△ WARNINGS found — review before deploying")
    else:            print("✗ BREAKING CHANGES found — do not replace production writer")
    sys.exit(0 if worst == 0 else (1 if worst == 2 else 2))

if __name__ == "__main__":
    main()

CheckWhat it catchesSeverityDuckDB function
Schema diff (Q1)Column removed, type changed, column addedBREAK / WARNparquet_schema()
Row count diff (Q2)Data loss, duplication, missing partitions, NC1 WAL wrong-dateBREAKread_parquet()
Null inflation (Q3)NC2 partial file, fields dropped from payload parsingBREAKparquet_metadata()
Timestamp range (Q4)NC1 WAL replay to wrong date partitionBREAKread_parquet()
Partition structure (Q5)Partition template changed, missing partitionsBREAKparquet_metadata()
Codec diff (Q6)Compression change (snappy → zstd)WARNparquet_metadata()
Noise & frozen sensors (Q8)Per-sensor delta-stddev noise floor, frozen sensors (range=0), flood candidates (>95% below hyst), fleet-wide hyst recommendationWARNread_parquet() with LAG()

Q3 (null inflation) and Q4 (timestamp range) work with footer-only reads via parquet_metadata() — fast enough to run on 400M+ row datasets in under a second on local NVMe. Q2 and Q4 require full scans — run against a 2-minute sample, not the entire production dataset.
