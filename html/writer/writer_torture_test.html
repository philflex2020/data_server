Overview — What Must Survive

Fixes under test
  • NC1 WAL replay writes to original date partition, not today
  • NC2 atomic temp+rename on flush, no partial .parquet files
  • NC3 fs::remove() uses error_code overload — never throws, flush thread stays up
  • NC4 pipe wake on overflow — no 60-second condvar stall
  • NM2 poll() replaces select() on health endpoint, FD > 1023 safe
  • NM4 SensorKey filtered to topic_kv_map columns only — no unbounded growth
Stress load
  • 2 sites × 4 racks × 4 modules × 52 cells = 1,664 cells
  • ~13,000 msgs/sec sustained into torture_broker
  • All 4 writers subscribe to the same topic space
  • Each writer produces independent parquet output to its own named volume
  • WAL enabled on all writers; flush interval 10 s; min_free_bytes 100 MB
Goal: every scenario runs to completion with zero data loss, no process crash, correct WAL replay to correct date partition, and no corrupt .parquet files on disk. A single BREAK in any scenario is a release blocker.

Setup Gotchas

Issues encountered setting up the stack on phil-256g and how they were resolved.

G1 — Arrow/Parquet .deb files not available in Ubuntu 22.04 repos

The Apache Arrow apt repository (apache.jfrog.io / packages.apache.org) returned 404 for Ubuntu 22.04. The libarrow2300 and libparquet2300 packages are not in the Ubuntu default repos.
Fix: copy the 5 .deb files directly from phil-dev's /var/cache/apt/archives/ to docker/torture/debs/ in the repo, then commit and push. The Dockerfile copies them into the build context and installs via apt-get install -y /tmp/debs/*.deb (which resolves runtime dependencies, unlike bare dpkg -i).

# From phil-dev — copy Arrow/Parquet debs into repo
mkdir -p docker/torture/debs
for pkg in libarrow2300 libarrow-dev libparquet2300 libparquet-dev apache-arrow-apt-source; do
    f=$(find /var/cache/apt/archives -name "${pkg}_*.deb" 2>/dev/null | head -1)
    [ -n "$f" ] && cp "$f" docker/torture/debs/ && echo "  $pkg OK"
done
# Then: git add docker/torture/debs/ && git commit && git push
# On phil-256g: git pull — debs arrive via git
G2 — Docker Compose v2 plugin not bundled with Docker 26 (Ubuntu repo)

Docker 26.1.3 installed from the Ubuntu repos does not include the docker compose v2 plugin. The old docker-compose standalone binary is separate and often absent.
Fix: install the plugin manually from GitHub releases.

# Install docker compose v2 plugin on phil-256g
mkdir -p ~/.docker/cli-plugins
curl -SL "https://github.com/docker/compose/releases/download/v2.27.0/docker-compose-linux-x86_64" \
     -o ~/.docker/cli-plugins/docker-compose
chmod +x ~/.docker/cli-plugins/docker-compose
docker compose version
# → Docker Compose version v2.27.0
G3 — stress_torture.yaml requires a projects: top-level key

stress_runner.py expects a projects: top-level key with a nested sites: / racks: / modules: structure (same format as config.yaml). A flat YAML (e.g. mqtt_host: broker at top level) causes KeyError: 'projects'.
Fix: use the nested format shown in docker/torture/configs/stress_torture.yaml (see the docker-compose section below for the mounted path).

G4 — cell_generator module not found inside stress container

stress_runner.py does sys.path.insert(0, os.path.join(_HERE, "..", "generator")) to import cell_generator. The container mounts the stress runner at /app, so the generator must be mounted at /app/../generator (i.e. /generator). Omitting this volume causes ModuleNotFoundError: No module named 'cell_generator'.
Fix: the compose file includes ../../source/generator:/app/../generator in the stress service volumes.

G5 — Dockerfile COPY path is relative to build context (repo root), not Dockerfile

The image is built with the repo root as the build context. COPY paths must be relative to the repo root, not relative to docker/torture/. COPY debs/ fails; the correct path is COPY docker/torture/debs/.

G6 — mosquitto healthcheck needs a working pub command

The broker healthcheck uses mosquitto_pub -h localhost -t hc -m 1 -q 0. This requires the mosquitto client tools to be present in the container (they are in eclipse-mosquitto:2.0 by default). All writer services depend on broker: condition: service_healthy, so the stress runner and writers will not start until the broker passes its health check.

fault_inject.sh

Actual running script at docker/torture/scripts/fault_inject.sh. The fault container has the Docker socket mounted and uses docker-cli from Alpine's apk to control sibling containers.

#!/usr/bin/env bash
# fault_inject.sh — torture test fault injector
# Runs inside the 'fault' container (has docker CLI + host docker socket)
#
# Env vars:
#   SCENARIO      broker_kill | disk_fill | chmod_flip | overflow | kill9 | all
#   CYCLE_SECONDS interval between fault injections (default 60)
#   TARGET        target writer container name (default torture_writer_d)

set -euo pipefail

SCENARIO="${SCENARIO:-all}"
CYCLE="${CYCLE_SECONDS:-60}"
TARGET="${TARGET:-torture_writer_d}"
LOG=/tmp/fault_inject.log

log() { echo "[$(date '+%H:%M:%S')] $*" | tee -a "$LOG"; }

# ── S1: broker_kill ──────────────────────────────────────────────────────
# Stops the broker for 30s to force WAL accumulation, then restarts.
# Writers should reconnect and replay WAL to the original date partition.
broker_kill() {
    log "S1 broker_kill: stopping torture_broker for 30s"
    docker stop torture_broker
    sleep 30
    docker start torture_broker
    sleep 10
    log "S1 broker_kill: broker restarted — writers should reconnect and replay WAL"
}

# ── S2: disk_fill ────────────────────────────────────────────────────────
# Fills the writer-c data volume to trigger the disk guard, then removes.
disk_fill() {
    log "S2 disk_fill: filling /data-c to trigger guard"
    AVAIL_KB=$(df /data-c | awk 'NR==2 {print $4}')
    FILL_KB=$(( AVAIL_KB - 512000 ))
    if [ "$FILL_KB" -gt 0 ]; then
        fallocate -l "${FILL_KB}k" /data-c/.disk_fill_blocker 2>/dev/null || \
            dd if=/dev/zero of=/data-c/.disk_fill_blocker bs=1M count=$(( FILL_KB / 1024 )) status=none
        log "S2 disk_fill: created ${FILL_KB}k blocker — writer-c should log LOW SPACE"
        sleep 15
        rm -f /data-c/.disk_fill_blocker
        log "S2 disk_fill: removed blocker — writer-c should resume"
    else
        log "S2 disk_fill: volume already nearly full (${AVAIL_KB}k avail) — skipping"
    fi
}

# ── S3: chmod_flip ───────────────────────────────────────────────────────
# Makes writer-c output dir temporarily unwritable during a flush window.
# NC2 (temp+rename) and NC3 (no-throw remove) should prevent corrupt files.
chmod_flip() {
    log "S3 chmod_flip: revoking write on /data-c for 500ms"
    chmod 555 /data-c
    sleep 0.5
    chmod 755 /data-c
    log "S3 chmod_flip: restored — checking for .tmp orphans"
    ORPHANS=$(find /data-c -name '*.tmp' 2>/dev/null | wc -l)
    log "S3 chmod_flip: orphan .tmp files = $ORPHANS (expect 0)"
}

# ── S4: overflow ─────────────────────────────────────────────────────────
# Pauses the target writer container briefly so MQTT messages pile up.
# NC4 pipe-wake should flush within 1s after resume.
overflow() {
    log "S4 overflow: pausing $TARGET for 5s to build buffer overflow"
    docker pause "$TARGET" || { log "S4 overflow: pause failed (container not running?)"; return; }
    sleep 5
    docker unpause "$TARGET"
    log "S4 overflow: unpaused — watching health endpoint for rapid flush"
    local before after port
    case "$TARGET" in
        *_a) port=8771 ;; *_b) port=8772 ;; *_c) port=8773 ;; *) port=8774 ;;
    esac
    before=$(curl -s "http://localhost:${port}/health" 2>/dev/null | grep -o '"flush_count":[0-9]*' | cut -d: -f2 || echo 0)
    sleep 3
    after=$(curl -s "http://localhost:${port}/health" 2>/dev/null | grep -o '"flush_count":[0-9]*' | cut -d: -f2 || echo 0)
    log "S4 overflow: flush_count before=$before after=$after (expect after > before within 3s)"
}

# ── S5: kill9 ────────────────────────────────────────────────────────────
# Sends SIGKILL to the writer process inside the target container.
# Docker restart policy brings it back; WAL should replay without duplicates.
kill9() {
    log "S5 kill9: sending SIGKILL to parquet_writer in $TARGET"
    docker exec "$TARGET" sh -c 'kill -9 $(pgrep parquet_writer)' 2>/dev/null || true
    sleep 5
    log "S5 kill9: container should have restarted — checking health"
    case "$TARGET" in
        *_a) port=8771 ;; *_b) port=8772 ;; *_c) port=8773 ;; *) port=8774 ;;
    esac
    curl -s "http://localhost:${port}/health" 2>/dev/null | \
        python3 -c "import sys,json; d=json.load(sys.stdin); print(f'  msgs_received={d[\"msgs_received\"]} wal_replay_rows={d[\"wal_replay_rows\"]}')" \
        2>/dev/null || log "S5 kill9: health endpoint not yet up (normal — still restarting)"
}

# ── dispatch ─────────────────────────────────────────────────────────────
run_one() {
    case "$1" in
        broker_kill) broker_kill ;;
        disk_fill)   disk_fill   ;;
        chmod_flip)  chmod_flip  ;;
        overflow)    overflow    ;;
        kill9)       kill9       ;;
        *) log "Unknown scenario: $1" ;;
    esac
}

log "fault_inject starting: SCENARIO=$SCENARIO CYCLE=${CYCLE}s TARGET=$TARGET"

if [ "$SCENARIO" = "all" ]; then
    SCENARIOS=(broker_kill disk_fill chmod_flip overflow kill9)
    i=0
    while true; do
        s="${SCENARIOS[$((i % ${#SCENARIOS[@]}))]}"
        log "── running $s ──"
        run_one "$s"
        log "── $s done — sleeping ${CYCLE}s ──"
        sleep "$CYCLE"
        i=$((i + 1))
    done
else
    while true; do
        log "── running $SCENARIO ──"
        run_one "$SCENARIO"
        log "── $SCENARIO done — sleeping ${CYCLE}s ──"
        sleep "$CYCLE"
    done
fi
The fault container installs docker-cli, bash, and util-linux (for fallocate) from Alpine apk at startup. The overflow scenario uses docker pause/unpause rather than tc netem — simpler and doesn't require NET_ADMIN capability. The disk_fill scenario targets the data-c volume which is also mounted into the fault container at /data-c.
