Overview

Lab server capacity
  • 256 GB RAM — run dozens of full topologies in parallel
  • Each topology: FlashMQ broker + writer.cpp + stress_runner = ~300 MB RSS
  • Comfortably fit 4 simultaneous writer instances at 80k msgs/sec each
  • Docker named volumes isolate output dirs per writer
  • All containers on a private supermicro bridge network
What must survive
  • NC1 — WAL replay writes to original date partition, not today
  • NC2 — atomic temp+rename on flush, no partial .parquet files
  • NC3 — remove() in destructor never throws, flush thread stays up
  • NC4 — pipe wake on overflow, no 60-second CV stall
  • NM2 — poll() replaces select(), FD > 1023 safe on health endpoint
  • NM4 — SensorKey includes writer_id, no cross-contamination
Goal: every scenario runs to completion with zero data loss, no process crash, correct WAL replay to correct date partition, and no corrupt .parquet files on disk. A single BREAK in any scenario is a release blocker.

fault_inject.sh

Place at docker/torture/fault_inject.sh. The fault-injector container runs this script. Set SCENARIO and WRITER env vars when launching the container.

#!/bin/sh
# fault_inject.sh — torture test fault injection driver
# Env vars:
#   SCENARIO      broker_kill | disk_fill | chmod_flip | overflow | kill9
#   WRITER        writer-a | writer-b | writer-c | writer-d
#   CYCLE_SECONDS seconds between fault cycles (default 60)
#   DATA_DIR      path to writer output volume (default /data/writer-a)

set -e

SCENARIO="${SCENARIO:-broker_kill}"
WRITER="${WRITER:-writer-a}"
CYCLE_SECONDS=${CYCLE_SECONDS:-60}
DATA_DIR="${DATA_DIR:-/data/writer-a}"
LOG="/tmp/fault_inject.log"

log() {
  echo "[$(date -u +%Y-%m-%dT%H:%M:%SZ)] [${SCENARIO}] $*" | tee -a "$LOG"
}

scenario_broker_kill() {
  log "Stopping flashmq container for 30s"
  docker stop flashmq
  sleep 30
  docker start flashmq
  log "flashmq restarted — waiting for reconnect"
  sleep 5
}

scenario_disk_fill() {
  # Fill the writer's data volume to within ~10MB of min_free_bytes
  FILL_FILE="${DATA_DIR}/__torture_fill__"
  AVAIL_KB=$(df -k "$DATA_DIR" | awk 'NR==2{print $4}')
  # Leave 8 MB free so we just cross the threshold
  FILL_KB=$(( AVAIL_KB - 8192 ))
  if [ "$FILL_KB" -gt 0 ]; then
    log "fallocate ${FILL_KB}KB into ${FILL_FILE}"
    fallocate -l "${FILL_KB}k" "$FILL_FILE"
    sleep 10
    log "Removing fill file, freeing space"
    rm -f "$FILL_FILE"
  else
    log "WARN: not enough space to fill — skipping"
  fi
}

scenario_chmod_flip() {
  # Remove write permission for 500ms during a flush window
  log "chmod 000 ${DATA_DIR}"
  chmod 000 "$DATA_DIR"
  sleep 0.5
  chmod 755 "$DATA_DIR"
  log "chmod 755 ${DATA_DIR} — window closed"
}

scenario_overflow() {
  # Throttle the flashmq container's network interface to simulate broker lag.
  # Adds 10s of artificial delay so the stress runner outruns the broker,
  # triggering NC4 pipe-wake overflow path.
  # Requires NET_ADMIN capability on the fault-injector container.
  IFACE="eth0"
  log "tc netem delay 10000ms on flashmq interface — overflow scenario"
  docker exec flashmq sh -c "tc qdisc add dev ${IFACE} root netem delay 10000ms 2>/dev/null || \
    tc qdisc change dev ${IFACE} root netem delay 10000ms"
  sleep 15
  docker exec flashmq sh -c "tc qdisc del dev ${IFACE} root 2>/dev/null || true"
  log "tc netem removed — throughput restored"
}

scenario_kill9() {
  # Kill the writer process inside its container; Docker restart=unless-stopped
  # brings it back automatically.  We do this repeatedly every CYCLE_SECONDS.
  WRITER_PID=$(docker exec "$WRITER" pgrep -x parquet_writer 2>/dev/null || true)
  if [ -n "$WRITER_PID" ]; then
    log "kill -9 parquet_writer PID=${WRITER_PID} in ${WRITER}"
    docker exec "$WRITER" kill -9 "$WRITER_PID"
  else
    log "WARN: parquet_writer not found in ${WRITER} — already restarting?"
  fi
  # Give Docker time to restart the container before next cycle
  sleep 5
}

log "fault_inject.sh started: SCENARIO=${SCENARIO} WRITER=${WRITER} CYCLE=${CYCLE_SECONDS}s"

while true; do
  case "$SCENARIO" in
    broker_kill)  scenario_broker_kill  ;;
    disk_fill)    scenario_disk_fill    ;;
    chmod_flip)   scenario_chmod_flip   ;;
    overflow)     scenario_overflow     ;;
    kill9)        scenario_kill9        ;;
    *)
      log "ERROR: unknown SCENARIO='${SCENARIO}'. Use: broker_kill disk_fill chmod_flip overflow kill9"
      exit 1
      ;;
  esac
  log "Cycle complete — sleeping ${CYCLE_SECONDS}s"
  sleep "$CYCLE_SECONDS"
done
The overflow scenario requires the fault-injector container to have cap_add: [NET_ADMIN] and the iproute2 package installed (add apk add iproute2 docker-cli to the Alpine image or use a custom image). The kill9 scenario requires /var/run/docker.sock mounted.

Validation Queries

Run these after any torture test scenario completes. All queries use DuckDB via .venv/bin/python3 from repo root. Replace path variables with the actual writer output dirs mounted from the named Docker volumes.

Query 1 — No corrupt files

Any file that parquet_metadata() fails to parse raises an exception. A clean result means every file is structurally valid.

import duckdb, glob, sys

con = duckdb.connect()
data_dir = "/data/writer-a"
files = glob.glob(f"{data_dir}/**/*.parquet", recursive=True)
corrupt = []
for f in files:
    try:
        con.execute(f"SELECT COUNT(*) FROM parquet_metadata('{f}')").fetchone()
    except Exception as e:
        corrupt.append((f, str(e)))

if corrupt:
    print(f"BREAK: {len(corrupt)} corrupt files")
    for path, err in corrupt:
        print(f"  {path}: {err}")
    sys.exit(1)
else:
    print(f"OK: all {len(files)} files are valid parquet")

Query 2 — WAL date correctness (S1 check)

The timestamp range for every row in a partition must fall within the date encoded in the directory name. A mismatch means NC1 WAL replay wrote to the wrong date.

import duckdb, pathlib, datetime

con = duckdb.connect()
data_dir = pathlib.Path("/data/writer-a")
mismatches = []

for date_dir in sorted(data_dir.glob("date=*")):
    date_str = date_dir.name.split("=")[1]   # e.g. "2026-03-21"
    parquet_glob = f"{date_dir}/**/*.parquet"
    row = con.execute(f"""
        SELECT
          MIN(timestamp)::DATE AS ts_min,
          MAX(timestamp)::DATE AS ts_max
        FROM read_parquet('{parquet_glob}', union_by_name=true)
    """).fetchone()
    if row and row[0] is not None:
        dir_date = datetime.date.fromisoformat(date_str)
        if row[0] != dir_date or row[1] != dir_date:
            mismatches.append((date_str, row[0], row[1]))

if mismatches:
    print("BREAK: WAL replay date mismatch (NC1 regression)")
    for d, lo, hi in mismatches:
        print(f"  dir={d}  ts_min={lo}  ts_max={hi}")
else:
    print("OK: all partition dates match their directory")

Query 3 — No duplicate rows

Compares total row count against count of distinct primary key tuples. Any gap means WAL was replayed more than once or message delivered twice.

import duckdb

con = duckdb.connect()
data_dir = "/data/writer-a"
result = con.execute(f"""
    SELECT
      COUNT(*)                                                        AS total_rows,
      COUNT(DISTINCT (timestamp, cell_id, site_id, rack_id, module_id)) AS distinct_rows,
      COUNT(*) - COUNT(DISTINCT (timestamp, cell_id, site_id, rack_id, module_id)) AS duplicates
    FROM read_parquet('{data_dir}/**/*.parquet', union_by_name=true)
""").fetchone()

total, distinct, dupes = result
print(f"total={total}  distinct={distinct}  duplicates={dupes}")
if dupes > 0:
    print("BREAK: duplicate rows detected — WAL replayed more than once?")
else:
    print("OK: no duplicates")

Query 4 — Row count per writer

All four writers subscribe to the same topic space and receive the same messages. Counts should match within 1% of each other. A large deviation indicates NM4 cross-contamination or a missed message batch.

import duckdb

con = duckdb.connect()
writers = ["writer-a", "writer-b", "writer-c", "writer-d"]
counts = {}

for w in writers:
    row = con.execute(f"""
        SELECT COUNT(*) FROM read_parquet('/data/{w}/**/*.parquet', union_by_name=true)
    """).fetchone()
    counts[w] = row[0]
    print(f"  {w}: {row[0]:,} rows")

vals = list(counts.values())
lo, hi = min(vals), max(vals)
pct_diff = (hi - lo) / hi * 100 if hi > 0 else 0
print(f"spread: {pct_diff:.3f}%")
if pct_diff > 1.0:
    print("BREAK: row count spread > 1% across writers")
else:
    print("OK: counts within 1%")

Query 5 — Zero nulls across all columns

Uses parquet_metadata() statistics. stats_null_count > 0 for any column in any row-group means a field was not written correctly — likely a serialisation bug triggered by the fault injection.

import duckdb

con = duckdb.connect()
data_dir = "/data/writer-a"
result = con.execute(f"""
    SELECT
      file_name,
      path_in_schema   AS column_name,
      SUM(stats_null_count) AS total_nulls
    FROM parquet_metadata('{data_dir}/**/*.parquet')
    WHERE stats_null_count > 0
    GROUP BY file_name, path_in_schema
    ORDER BY total_nulls DESC
    LIMIT 20
""").fetchall()

if result:
    print(f"BREAK: null values found in {len(result)} column(s)")
    for row in result:
        print(f"  {row[0]}  col={row[1]}  nulls={row[2]}")
else:
    print("OK: zero nulls in all parquet column statistics")

Quick-start

1. Build the writer image

# From repo root on the Supermicro host
docker build -t parquet-writer source/parquet_writer_cpp/

2. Create per-writer config files

mkdir -p docker/torture/configs

for w in a b c d; do
  PORT=$((8770 + $(echo a b c d | tr ' ' '\n' | grep -n "^${w}$" | cut -d: -f1)))
  cat > docker/torture/configs/writer_${w}.yaml <<EOF
mqtt_host: flashmq
mqtt_port: 1883
output_dir: /data/writer-${w}
health_port: ${PORT}
writer_id: writer-${w}
wal_dir: /data/writer-${w}/.wal
min_free_bytes: 104857600
flush_interval_seconds: 10
EOF
done

3. Start all services

docker-compose -f docker/torture/docker-compose.yml up -d

# Verify all containers are running
docker-compose -f docker/torture/docker-compose.yml ps

4. Inject a specific fault scenario

# S1: broker kill+restart on writer-a (default CYCLE=60s)
SCENARIO=broker_kill WRITER=writer-a CYCLE_SECONDS=60 \
  docker-compose -f docker/torture/docker-compose.yml run fault-injector

# S2: disk fill on writer-b
SCENARIO=disk_fill WRITER=writer-b DATA_DIR=/data/writer-b \
  docker-compose -f docker/torture/docker-compose.yml run fault-injector

# S3: chmod flip on writer-c
SCENARIO=chmod_flip WRITER=writer-c DATA_DIR=/data/writer-c \
  docker-compose -f docker/torture/docker-compose.yml run fault-injector

# S4: overflow (tc netem) on writer-a via broker throttle
SCENARIO=overflow WRITER=writer-a \
  docker-compose -f docker/torture/docker-compose.yml run fault-injector

# S5: kill -9 writer-d every 60s
SCENARIO=kill9 WRITER=writer-d CYCLE_SECONDS=60 \
  docker-compose -f docker/torture/docker-compose.yml run fault-injector

5. Watch health endpoints live

watch -n2 'for p in 8771 8772 8773 8774; do
  echo -n "port $p: "
  curl -s localhost:$p/health | python3 -m json.tool
done'

6. FD stress test (S6)

# Requires apache bench — 2000 concurrent connections, 4000 total requests
ab -c 2000 -n 4000 http://localhost:8771/health

# Watch for crashes in the writer-a log
docker logs -f writer-a 2>&1 | grep -E '(CRASH|select|poll|accept|BREAK)'

7. Run validation after a test

.venv/bin/python3 scripts/torture_validate.py /data/writer-a /data/writer-b /data/writer-c /data/writer-d

8. S8 — 24-hour soak (all faults on 60 s cycle)

# Bring up a dedicated soak stack and rotate all four fault types across writers
for scenario in broker_kill disk_fill chmod_flip overflow; do
  SCENARIO=$scenario WRITER=writer-a CYCLE_SECONDS=60 \
    docker-compose -f docker/torture/docker-compose.yml run -d \

# Monitor fault log
docker exec fault-injector tail -f /tmp/fault_inject.log
